用戶端偵測

認證程序的起始步驟就是識別發出 HTTP(S) 請求的用戶端類型。此 Access Manager 功能稱為用戶端偵測。URL 資訊用於擷取用戶端的特徵。根據這些特性，會傳回適當的認證頁面。例如，使用 Netscape 瀏覽器請求網頁時，Access Manager 7.1 會顯示 HTML 登入頁面。一旦使用者通過驗證，用戶端類型 (Netscape 瀏覽器) 會新增至階段作業記號。用戶端偵測服務中定義的屬性為全域屬性。

用戶端類型

為了偵測用戶端類型，Access Manager 需要識別它們的識別特徵。這些特徵可識別用戶端資料格式的所有支援類型的內容。此屬性可讓您透過 [用戶端管理員] 介面修改用戶端資料。若要存取 [用戶端管理員]，請按一下 [編輯] 連結。依預設，Access Manager 包含以下用戶端類型：

  • HDML

  • HTML

  • JHTML

  • VoiceX

  • WML

  • XHTML

  • cHTML

  • iHTML

如需這些用戶端類型的說明， 請參閱 http://docs.sun.com/app/docs/coll/1483.1。

用戶端管理程式

[用戶端管理程式] 為列出基本用戶端、樣式和關聯內容的介面，它可讓您新增和配置裝置。基本用戶端類型在 [用戶端管理員] 頂部列出。這些用戶端類型包含屬於此用戶端類型的所有裝置可繼承的預設特性。

用戶端類型

樣式設定檔：[用戶端管理員] 會將用戶端類型清單中的所有可用用戶端 (包括基本用戶端類型本身) 分組。對於每一個用戶端，您可以按一下裝置名稱來修改用戶端特性。這些特性則顯示在 [用戶端編輯程式] 視窗中。若要編輯這些特性，請從下拉式清單中選取以下類別：

硬體平台

包含裝置的硬體屬性，如顯示大小、支援的字元集等。

軟體平台

包含裝置的應用程式環境的屬性、作業系統的屬性以及安裝軟體的屬性。

網路特性

包含描述網路環境 (包括支援的載送程式) 的特性。

瀏覽器 UA

包含與在此裝置上執行的瀏覽器使用者代理程式相關的屬性。

WapCharacteristics

包含此裝置支援的無線應用程式協定 (WAP) 環境的特性。

PushCharacteristicNames

包含此裝置支援的 WAP 環境的特性。

其他特性

包含此裝置支援的無線應用程式協定 (WAP) 環境的特性。

備註 – 如需特定的特性定義，請參閱 Open Mobile Alliance Ltd. (OMA) 無線應用程式協定，版本 20-Oct-2001，網址為 http://www1.wapforum.org/tech/terms.asp?doc=WAP-248-UAProf-20011020-a.pdf。

為了存取此文件，您必須先註冊 WAP Forum™。如需相關資訊，請造訪 http://www.wapforum.org/faqs/index.htm。

預設用戶端類型

此屬性定義從 [用戶端類型] 屬性的用戶端類型清單中導出的預設用戶端類型。預設值為 genericHTML。

用戶端偵測類別

此屬性定義路由所有用戶端偵測請求的用戶端偵測類別。此屬性傳回的字串應該與 [用戶端類型] 屬性中列出的某種用戶端類型相符。預設用戶端偵測類別為 com.sun.mobile.cdm.FEDIClientDetector。Access Manager 也包含 com.iplanet.services.cdm.ClientDetectionDefaultImpl。

用戶端偵測

啟用用戶端偵測。如果啟用了用戶端偵測 (預設值)，則會透過用戶端偵測類別屬性中指定的類別路由每個請求。依預設，會啟用用戶端偵測功能。如果未選取此屬性，則 Access Manager 假定用戶端是 genericHTML，並可透過 HTML 瀏覽器存取。
