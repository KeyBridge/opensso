建立新代理程式

建立新代理程式也稱為建立新代理程式設定檔。您可以在 Federated Access Manager 管理主控台中執行這項作業。這項作業適用於 Web 代理程式、J2EE 代理程式和 2.2 代理程式。不過，某些個別步驟不適用 2.2 代理程式。

  1. 按一下 [配置] 標籤下的 [代理程式]。
  2. 如有必要，請選取適當代理程式類型的標籤。
    • Web 代理程式：

      [Web] 標籤用於 Web 代理程式，且為預設選項。不需要再進行選取。

    • J2EE 代理程式：

      [J2EE] 標籤用於 J2EE 代理程式。如果適用，請按一下此標籤。

    • 2.2 代理程式：

      [2.2 代理程式] 標籤用於「策略代理程式 2.2」軟體集中的代理程式。如果適用，請按一下此標籤。

  3. 在 [代理程式] 區段中，按一下 [新建]。
  4. 在 [名稱] 欄位中，輸入新代理程式設定檔的名稱。
  5. 輸入並確認 [密碼]。

    注意 – 此密碼必須與您執行 agentadmin 程式安裝代理程式時，在指定的代理程式設定檔密碼檔中輸入的密碼相同。

  6. 選取適當的配置：[本機] 或 [集中式] (不適用於 2.2 代理程式)。

    此外，選取 [本機] 時，與此代理程式相關的屬性只能從代理程式中的檔案編輯。這些屬性無法從主控台編輯。

  7. 在 [伺服器 URL] 欄位中，輸入 Federated Access Manager 伺服器 URL (不適用於 2.2 代理程式)。

    例如：

    http://fam.example.com:8080/fam
  8. 在 [代理程式 URL] 欄位中，輸入代理程式應用程式 agentapp 的 URL (不適用於 2.2 代理程式)。

    例如：

    http://fam.example.com:8090/agentapp
  9. 按一下 [建立]。

    主控台會建立代理程式設定檔，並且再次顯示代理程式 (WebAgent、J2EEAgent 或 2.2_Agent) 頁面，其中含有新代理程式設定檔的連結。

    若要進行額外的代理程式設定檔配置，請按一下此連結，以顯示 [編輯] 代理程式頁面。
