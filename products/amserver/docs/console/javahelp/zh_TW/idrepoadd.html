Access Manager 資料存放區類型

在 [Access Manager 資料存放區選取] 畫面中，可以建立下列資料存放區類型：

Access Manager 儲存庫外掛程式

此資料存放區類型位於 Sun Java System Directory Server 實例中，並擁有 Access Manager 資訊樹狀結構。此資料存放區類型使用不屬於 LDAP 版本 3 規格的 Directory Server 功能 (如角色及服務類別)，並與先前版本的 Access Manager 相容。

如需屬性定義的相關資訊，請參閱Access Manager 儲存庫屬性。

Active Directory

此資料存放區類型使用 LDAP 版本 3 規格向 Microsoft Active Directory 的實例寫入識別資料。

如需屬性定義的相關資訊，請參閱Active Directory 屬性。

平面檔案儲存庫

此儲存庫可讓您在 Access Manager 的本機安裝實例上以平面 DIT 結構儲存資料及識別，而不必建立個別的資料存放區。它通常用來測試或驗證概念部署。

如需屬性定義的相關資訊，請參閱平面檔案儲存庫屬性。

通用 LDAPv3

此資料存放區類型可讓您向任何與 LDAPv3 相容的資料庫寫入識別資料。如果您使用的 LDAPv3 資料庫不支援持續搜尋，則無法使用快取功能。

如需屬性定義的相關資訊，請參閱通用 LDAPv3 屬性。

具有 Access Manager 模式的 Sun Directory Server

此資料存放區類型位於 Sun Java System Directory Server 實例中，並擁有 Access Manager 資訊樹狀結構。它與 Access Manager 儲存庫外掛程式的不同之處在於，它有更多配置屬性可讓您更好地自訂資料存放區。

如需屬性定義的相關資訊，請參閱具有 Access Manager 模式的 Sun Directory Server 屬性。
