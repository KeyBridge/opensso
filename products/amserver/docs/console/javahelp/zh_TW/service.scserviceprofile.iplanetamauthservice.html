核心

此模組是 Access Manager 認證服務的一般配置基礎。它必須完成註冊及配置，才能使用任一個特定的認證模組實例。它可讓管理員定義預設值，當未在 Access Manager 認證模組中特別設定值時，將挑選這些值。核心屬性為全域和範圍屬性。這些屬性包括：

  • 可插接式認證模組類別

  • 用戶端支援的認證模組

  • LDAP 連接儲存區大小

  • 預設 LDAP 連接儲存區大小

  • 使用者設定檔

  • 管理員認證配置

  • 使用者設定檔動態建立預設角色

  • 永久性 Cookie 模式

  • 永久性 Cookie 最長時間

  • 別名搜尋屬性名稱

  • 預設認證語言環境

  • 組織認證配置

  • 登入失敗封?模式

  • 登入失敗封?計數

  • 登入失敗封?間隔時間

  • 接收封?通知的電子郵件地址

  • N 次失敗後警告使用者

  • 登入失敗封?持續時間

  • 封?屬性名稱

  • 封?屬性值

  • 預設成功登入 URL

  • 預設失敗登入 URL

  • 認證處理後類別

  • 產生使用者 ID 模式

  • 可插接式使用者名稱產生器類別

  • 識別類型

  • 可插接式使用者狀態事件類別

  • 在資料存放區中儲存無效嘗試

  • 基於模組的認證

  • 預設認證層級

可插接式認證模組類別

指定 Access Manager 平台內配置的所有範圍均可以使用的認證模組之 Java 類別。您可以透過實作 AMLoginModule SPI 或 JAAS LoginModule SPI 寫入自訂認證模組。如需更多資訊，請參閱「Access Manager Developer's Guide」。若要定義新的服務，此欄位必須採用指定每個新認證服務之完整類別名稱 (包括套裝軟體名稱) 的文字字串。

用戶端支援的認證模組

指定特定用戶端支援的認證模組的清單。格式如下所示：

clientType | module1,module2,module3

此屬性在啟用了用戶端偵測時有效。

LDAP 連接儲存區大小

指定在特定 LDAP 伺服器與連接埠上使用的最小與最大連線池。此屬性僅用於 LDAP 與成員身份認證服務。格式如下所示：

host:port:min:max

備註 – 此連線池不同於 serverconfig.xml 中配置的 SDK 連線池。

預設 LDAP 連接儲存區大小

設定與所有 LDAP 認證模組配置一同使用的連線池預設最小值與最大值。如果 LDAP 連線池大小屬性中存在主機與連接埠的項目，則不會使用 LDAP 預設連線池大小中的最小值與最大值設定。

使用者設定檔

此選項可讓您為使用者設定檔指定選項。選項如下：

必需

此選項指定，對於成功認證，使用者在安裝有 Access Manager 的本機 Directory Server 中需要有設定檔，認證服務才可以發行 SSOToken。

動態

此選項指定，對於成功認證，如果尚不存在使用者設定檔，認證服務將建立一個使用者設定檔。然後將發行 SSOToken。將在安裝有 Access Manager 的本機 Directory Server 中建立使用者設定檔。

隨使用者別名動態變化

此選項指定，對於成功認證，認證服務將使用使用者別名清單屬性來建立使用者設定檔。

忽略

此選項指定，對於成功認證，認證服務不需要使用者設定檔便可以發行 SSOToken。

管理員認證配置

僅定義管理員的認證服務。如果需要管理員的認證模組與使用者的認證模組有所不同，則可以使用此屬性。此屬性中配置的模組將在存取 Access Manager 主控台時被挑選出來。例如：

http://servername.port/console_deploy_uri

使用者設定檔動態建立預設角色

此欄位指定分配給新使用者的角色，當透過使用者設定檔選取 [動態建立] 時，會建立這些新使用者的設定檔。此欄位沒有預設值。管理員必須指定將分配給新使用者的角色之 DN。

備註 – 指定的角色必須位於欲配置認證之範圍下。此角色可以是 Access Manager 角色或 LDAP 角色，但不能是已篩選的角色。

若要自動指定特定服務給使用者，您必須在使用者設定檔中配置 [必須的服務] 屬性。

永久性 Cookie 模式

此選項確定使用者是否可以重新啟動瀏覽器，並且仍然返回至其經過認證的階段作業。可以透過啟用 [啟用永久性的 Cookie 模式] 保留使用者階段作業。啟用了 [啟用永久性的 Cookie 模式] 時，使用者階段作業在其永久性的 Cookie 過期或者該使用者明確登出後才會過期。過期時間是在 [永久性 Cookie 最長時間] 中指定。預設值是未啟用永久性的 Cookie 模式，並且認證服務僅使用記憶體 Cookie。

備註 – 用戶端必須使用登入 URL 中的 iPSPCookie=yes 參數，明確請求永久性 Cookie。

永久性 Cookie 最長時間

指定永久性 Cookie 多長時間後會過期。這一間隔時間在成功認證使用者階段作業後開始。最大值為 2147483647 (以秒計算的時間)。欄位將接受小於最大值的任何整數值。

別名搜尋屬性名稱

使用者成功認證後，將擷取使用者設定檔。如果依據首選 LDAP 屬性執行的搜尋，無法找到相符的使用者設定檔，則此欄位會指定另一個要從中搜尋的 LDAP 屬性。此屬性將主要在從認證模組傳回的使用者識別不同於 [使用者命名屬性] 中指定的識別時使用。例如，RADIUS 伺服器可能會傳回 abc1234，但是使用者名稱卻為 abc。此屬性沒有預設值。

此欄位將接受任何有效的 LDAP 屬性 (例如，cn)。

預設認證語言環境

指定認證服務將使用的預設語言子類型。預設值為 en_US。請參閱支援的語言環境，以取得有效的語言子類型清單。

為了使用其他語言環境，必須首先建立此語言環境的所有認證範本。然後必須為這些範本建立新目錄。如需更多資訊，請參閱「管理指南」中的「登入 URL 參數」。

組織認證配置

設定組織的認證模組。預設認證模組為 LDAP。

登入失敗封?模式

指定使用者在首次認證嘗試失敗後是否可以再次嘗試。選取此屬性會啟用封?，使用者僅有一次認證的機會。依預設，封?功能是停用的。此屬性同與封?相關的屬性以及通知屬性配合使用。

登入失敗封?計數

定義在登入失敗封?間隔時間所定義的時間間隔內，使用者在封?之前可以嘗試進行認證的次數。

登入失敗封?間隔時間

定義兩次登入嘗試失敗之間的時間 (以分鍾為單位)。如果某次登入失敗，並且在封?間隔時間內再次登入失敗，則增加封?計數。否則重設封?計數。

接收封?通知的電子郵件地址

指定將接收使用者封?通知的電子郵件地址。若要將電子郵件通知傳送至多個位址，請使用空格分隔每個電子郵件地址。如果不是英文語言環境，其格式為：

email_address|locale|charset

N 次失敗後警告使用者

指定在 Access Manager 傳送使用者將被封?的警告訊息之前，可以發生的認證失敗次數。

登入失敗封?持續時間

啟用記憶體鎖定。依預設，封?機制將使封?屬性名稱中定義的使用者設定檔處於非使用中 (登入失敗後)。如果 [登入失敗封?持續時間] 的值大於 0，則其記憶體鎖定和使用者帳戶將被封?一段指定的時間 (分鐘)。

封?屬性名稱

指定要被設定為封?的所有 LDAP 屬性。還必須變更封?屬性值中的值以啟用此屬性名稱的封?。依預設，Access Manager 主控台中的 [封?屬性名稱] 為空。當使用者被封?且 [登入失敗封?持續時間] 設定為 0 時，預設實作值為 inetuserstatus (LDAP 屬性) 和 inactive。

封?屬性值

此屬性指定啟用還是停用封?屬性名稱中定義之屬性的封?。依預設，inetuserstatus 的值設定為非使用中。

預設成功登入 URL

此欄位接受多值清單，這些值指定認證成功後，會將使用者重新導向至哪一個 URL。此屬性格式為 clientType|URL，但您僅能指定假設為 HTML 預設類型的 URL 值。預設值為 /amserver/console。

預設失敗登入 URL

此欄位接受多值清單，這些值指定認證成功後，會將使用者重新導向至哪一個 URL。此屬性格式為 clientType|URL，但您僅能指定假設為 HTML 預設類型的 URL 值。

認證處理後類別

指定 Java 類別名稱，用於自訂登入成功或失敗的認證後程序。範例：

com.abc.authentication.PostProcessClass

Java 類別必須實作以下 Java 介面：

com.sun.identity.authentication.spi.AMPostAuthProcessInterface

此外，您必須將此類別所在位置的路徑加入到 Web Server 的 [Java 類別路徑] 屬性中。

產生使用者 ID 模式

成員身份認證模組使用此屬性。如果啟用了此屬性欄位，則成員身份模組能夠在自行註冊過程中，產生特定使用者的多個使用者 ID (如果使用者 ID 已經存在)。這些使用者 ID 是從可插接式使用者名稱產生器類別中指定的 Java 類別產生的。

可插接式使用者名稱產生器類別

指定當使用 [啟用產生 UserID 模式] 時產生使用者 ID 所使用的 Java 類別名稱。

識別類型

列出 Access Manager 將搜尋的識別類型。

可插接式使用者狀態事件類別

延伸認證 SPI 以為使用者於認證程序中之狀態變更提供回呼機制。支援下列狀態的變更：

帳號封?

帳號封?事件可用於任何認證模組。這些特性可以透過登入失敗封?模式屬性來配置。

密碼變更

僅可用於 LDAP 認證模組類型，因為密碼變更僅可用於該模組。

在資料存放區中儲存無效嘗試

如果啟用的話，此屬性允許在多個 Access Manager 實例共用的識別儲存庫中共用登入失敗嘗試。例如，若用於特定部署的識別儲存庫是 Directory Server，則無效的嘗試會儲存在 sunAMAuthInvalidAttemptsData (屬於 sunAMAuthAccountLockout 物件類別)。資料的格式會儲存為：

<InvalidPassword><InvalidCount></InvalidCount><LastInvalidAt></LastInvalidAt><LockedoutAt></LockedoutAt><ActualLockoutDuration></ActualLockoutDuration></InvalidPassword>

對於每一個使用者，此資訊皆在 Directory Server 中維護。當發生無效的嘗試時，<InvalidCount> 即會增加。

基於模組的認證

如果啟用的話，此屬性允許使用者透過基於模組的認證進行認證。如果未啟用此屬性，不允許基於模組的登入。使用 module=< module_instance_name> 進行的所有登入嘗試都會導致登入失敗。

預設認證層級

認證層級值指示信任認證的程度。使用者進行認證後，此值便會儲存在階段作業的 SSO 記號中。SSO 記號呈現給使用者要存取的應用程式時，該應用程式可以使用儲存的值以確定此層級是否達到了允許使用者存取的層級。如果儲存在 SSO 記號中的認證層級不滿足最小值需求，應用程式可以提示使用者透過具有較高認證層級的服務重新進行認證。

應該在範圍的特定認證範本中設定認證層級。只在未於 [認證層級] 欄位中為特定範圍的認證範本指定認證層級時，才會套用這裡描述的 [預設認證層級] 值。[預設認證層級] 預設值為 0。(Access Manager 並不使用此屬性中的值，而是由可以選擇使用它的任何外部應用程式使用。)
