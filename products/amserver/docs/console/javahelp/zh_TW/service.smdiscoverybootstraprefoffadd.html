配置啟動程式的資源提供

啟動程式只允許一個資源提供；依預設，此提供包含有關探索服務的資訊。如果已定義資源提供，您可以按一下其名稱來修改屬性。您也可以勾選名稱旁的核取方塊，然後刪除現有的資源提供。若要配置新的資源提供，請按一下 [新建]。

  1. 按一下 [新建] 以顯示新的對映程式屬性，或按一下已配置的對映程式名稱以修改它。

  2. 輸入資源提供的選擇性說明。

  3. 根據下列資訊提供 [服務實例] 屬性的值：

    服務類型

    此屬性取值為一個 URI，它定義了資源提供執行的服務類型。例如，urn:liberty:disco:2003-08。

    備註 – 建議此 URI 與該服務的摘要 WSDL 描述之 targetNamespace URI 相同。

    提供者 ID

    此屬性取值為服務實例提供者的 URI。例如，http://daiquiri.sun.com:80/amserver/Liberty/disco。

    服務說明

    此屬性會在不同的協定終點定義執行中的 Web 服務。服務實例的相關資訊需要在各種環境中進行通訊。例如，在此規格中定義的探索服務是識別服務，它會提供資源提供的列舉 (每一個都包括服務實例)。請參閱配置服務描述。

  4. 根據下列資訊提供 [資源提供選項] 屬性的值：

    選項

    如果服務沒有可供資源提供使用的選項，請勾選此核取方塊。這些選項提供提示給潛在的請求者，指出某些資料或作業對於特殊的資源提供是否可用。例如，可以提供一個選項，陳述可以使用住家連絡人資訊。

    選項清單

    此屬性包含服務實例的選項清單。選項定義為 URI。可能的 URI 集通常是由服務類型標準化。

  5. 根據下列資訊啟用指令：

    描述 ID 參考：GenerateBearerToken

    所有支援的指令都可能包含描述性參考。如果未對指令啟用 ID 參考屬性，將採用該指令以套用至資源提供中所提供的所有認證機制。如果對指令啟用了 ID 參考屬性，它「必須」包含 ID 參考的清單，指向與指令相關聯的認證機制。如果屬性存在，則「必須」採用指令，並僅套用至 ID 參考清單中指向的那些說明。如果某些指令與某些安全機制不相容，這樣做可能非常有用。
