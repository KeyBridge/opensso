Web 服务客户机属性

Web 服务客户机代理配置文件描述用于保护 Web 服务客户机的 Web 服务请求的配置。对每个 Web 服务客户机而言，Web 服务的名称必须唯一。

常规

以下“常规”属性定义 Web 服务客户机的基本属性：

组

可通过“组”机制来定义相似代理类型的集合。组必须在将特定代理纳入集合之前进行定义。

密码

定义 Web 服务客户机属性的密码。

密码确认

确认密码。

状态

定义 Web 服务客户机是否参与保护外部服务请求。默认情况下，该属性设置为激活，即参与保护。

通用标识符

列出用于唯一定义 Web 服务客户机代理的基本 LDAP 属性。

安全

以下属性定义 Web 服务客户机的安全属性：

安全机制

定义用于保护 Web 服务客户机请求的安全证书类型。可选择以下安全类型：

  • 匿名 — 匿名安全机制不包含安全证书。

  • LibertyDiscoverySecurity — 使用基于 Liberty 的安全令牌来保护请求。

  • SAML-HolderOfKey — 使用 SAML 1.1 声明类型 Holder-Of-Key 来保护请求。

  • SAML-SenderVouches — 使用 SAML 1.1 声明类型 Sender Vouches 来保护请求。

  • SAML2–HolderOfKey — 使用 SAML 2.0 声明令牌类型 Holder-Of-Key 来保护请求。

  • SAML2–SenderVouches — 使用 SAML 2.0 声明令牌类型 Sender Vouches 来保护请求。

  • STSSecurity — 定义 Web 服务客户机从安全令牌服务获取安全令牌。

  • UserNameToken — 使用用户名令牌来保护 Web 服务客户机请求。

  • UserNameToken-Plain — 使用带有明文密码的用户名令牌来保护 Web 服务请求。

  • X509Token — 使用 X509 证书来保护 Web 服务请求。

STS 配置

当 Web 服务客户机使用安全令牌服务 (Security Token service, STS) 时，启用该属性。该配置描述用于保护 STS 服务请求的 STS 代理配置文件列表。

搜索配置

当为搜索服务安全启用 Web 服务客户机时，启用该属性。该配置描述用于保护向搜索服务所作请求的搜索代理配置文件列表。

需要进行用户验证

启用后，该属性定义服务客户机保护页面要求用户经过验证才能获取访问权限。

保留消息中的安全头

启用后，该属性定义为 Web 服务客户机保留 SOAP 安全头，以进行进一步处理。

Liberty 服务类型 URN

URN（Universal Resource Name，通用资源名称）描述 Web 服务客户机用于服务查找的 Liberty 服务类型。

用户令牌的证书

该属性表示 Web 服务客户机用来生成用户名安全令牌的用户名/密码共享密钥。

签署和加密

以下属性定义 Web 服务安全的签署和加密配置：

请求是否已签名

启用后，Web 服务客户机使用给定的令牌类型签署请求。

请求标头是否已加密

启用后，将加密 Web 服务客户机的安全标头。

请求是否已加密

启用后，将启用 Web 服务客户机请求。

响应签名是否已验证

启用后，会验证 Web 服务响应签名。

响应是否已解密

启用后，将解密 Web 服务响应。

密钥库

以下属性配置用于证书存储和检索的密钥库：

Web 服务提供者的公钥别名

该属性定义用于加密 Web 服务请求的公共证书密钥别名。

私钥别名

该属性定义用于签署/解密 Web 服务响应的私有证书密钥别名。

密钥存储库用法

该配置定义是使用默认的密钥库，还是使用自定义的密钥库。必须为自定义密钥库定义以下值：

  • 密钥库位置

  • 密钥库密码

  • 密钥密码

端点

以下属性定义 Web 服务端点：

Web 服务端点

该属性定义 Web 服务客户机发送请求的 Web 服务端点。
