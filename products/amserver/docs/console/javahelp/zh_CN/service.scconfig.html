配置属性

“配置”页面包含用于配置 Access Manager 默认服务的所有属性。这些属性包含 Access Manager 服务，且被分为以下类型之一（某些服务可能有多种类型）：

全局 – 适用于整个 Access Manager 配置。由于全局属性的目的在于自定义 Access Manager 应用程序，因此此类属性不能直接应用到用户、角色或领域。

领域 – 领域属性仅指定给领域。没有与领域属性相关的对象类。由于验证是在领域级别而不是在子树或用户级别完成的，因此验证服务中列出的属性被定义为领域属性。

动态 – 指定给 Access Manager 的已配置角色或领域。当指定角色给用户，或在领域中创建用户时，动态属性将变为用户的一个特征。

用户 – 直接指定给每位用户。他们不是从角色或领域那里继承而来，通常对于每个用户都不同。

您可以修改的配置属性包括：

  • 验证

  • 控制台属性

  • 全局属性

  • 系统属性
