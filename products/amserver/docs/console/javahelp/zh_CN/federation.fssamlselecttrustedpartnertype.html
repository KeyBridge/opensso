可信赖的伙伴：选择伙伴类型和配置文件

此属性定义了即将与 Access Manager 进行通信的任何可信赖伙伴（需远程访问装有 Access Manager 的服务器）。

注 – 必须预先在可信赖的伙伴站点与“站点标识符”属性中配置的一个或多个站点之间安排某种信赖关系。

配置可信赖伙伴的第一步是确定伙伴在信赖关系中的角色。可信赖伙伴可以是源站点（生成单点登录声明）或目标站点（使用单点登录声明）。例如，如果该伙伴是源站点，则根据它发送声明的方式来配置此属性。如果该伙伴是目标站点，则根据它接收声明的配置文件来配置此属性。以下是配置可信赖伙伴的步骤的第一部分。起点是联合下的 "SAML" 屏幕。

注 – 要编辑或复制可信赖伙伴配置文件的属性，请在已配置的可信赖伙伴名称旁边的“操作”列中单击相应的按钮。

  1. 通过选择与其通信所使用的相应配置文件，为您要配置的伙伴站点选择角色（目标站点或源站点）。

    可为目标站点、源站点或两者选择 Web 浏览器辅件配置文件或 Web 浏览器 Post 配置文件，或者仅为目标站点选择 SOAP 查询。所做的选择指明在以下步骤中需要对哪些属性进行配置。

    注 – 如果要修改现有的可信赖伙伴，请单击“编辑”，更改伙伴站点的角色。

  2. 单击“下一步”。

  3. 要完成配置，请参见可信赖的伙伴：配置可信赖的伙伴属性。
