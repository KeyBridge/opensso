创建新代理

创建新代理也称为创建新的代理配置文件。您可以在 Federated Access Manager 管理控制台中执行该任务。该任务适用于 Web 代理、J2EE 代理和 2.2 代理。但是，某些个别步骤不适用于 2.2 代理。

  1. 单击“配置”选项卡下的“代理”。
  2. 如有必要，选择适当代理类型的选项卡。
    • Web 代理：

      "Web" 选项卡适用于 Web 代理且为默认选项。无需再次选择。

    • J2EE 代理：

      "J2EE" 选项卡适用于 J2EE 代理。如果适用，单击该选项卡。

    • 2.2 代理

      “2.2 代理”选项卡适用于“策略代理 2.2”软件集中的代理。如果适用，单击该选项卡。

  3. 在“代理”部分中单击“新建”。
  4. 在“名称”字段中输入新代理配置文件的名称。
  5. 输入“密码”并确认。

    注意 – 此密码必须与您运行 agentadmin 程序以安装代理时在指定的代理配置文件密码文件中输入的密码相同。

  6. 选择适当的配置：“本地”或“集中”（不适用于 2.2 代理）。

    此外，如果选择“本地”，则只能从代理的文件中编辑与此代理相关的属性。而无法从控制台编辑它们。

  7. 在“服务器 URL”字段中，输入 Federated Access Manager 服务器 URL（不适用于 2.2 代理）。

    例如：

    http://fam.example.com:8080/fam
  8. 在“代理 URL”字段中，输入代理应用程序 agentapp 的 URL（不适用于 2.2 代理）。

    例如：

    http://fam.example.com:8090/agentapp
  9. 单击“创建”。

    控制台创建代理配置文件，并再次显示代理（WebAgent、J2EEAgent 或 2.2_Agent）页面，其中包含指向新代理配置文件的链接。

    要对代理配置文件进行其他配置，单击该链接显示“编辑”代理页面。
