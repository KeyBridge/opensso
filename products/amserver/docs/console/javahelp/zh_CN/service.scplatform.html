平台

“平台”服务是可将其他服务器添加到 Access Manager 配置的位置，也是其他应用于 Access Manager 应用程序顶层的选项。平台服务属性是全局属性。这些属性包括：

  • 站点名称

  • 实例名称

  • 平台语言环境

  • Cookie 域

  • 登录服务 URL

  • 注销服务 URL

  • 可用的语言环境

  • 客户机字符集

站点名称

命名服务在初始化时读取该属性。该列表用后端 Access Manager 服务器的负载平衡器或用于负载平衡的 SRA 的端口号唯一标识 FQDN。如果列表中不存在服务 URL 请求中指定的主机，则命名服务将拒绝请求。此属性中应仅使用命名服务协议。参见创建新的站点名称。

实例名称

命名服务在初始化时读取该属性。该列表包含单个 Access Manager 配置中的 Access Manager 会话服务器。例如，如果安装了两个 Access Manager，且二者应作为一个 Access Manager 工作，则必须将它们都包含在该列表中。如果列表中不存在服务 URL 请求中指定的主机，则命名服务将拒绝请求。此属性中应仅使用命名服务协议。参见创建新的实例名称。

平台语言环境

平台语言环境的值是安装 Access Manager 时使用的默认语言子类型。验证、记录和管理服务是用该默认值的语言进行管理的。默认值为 en_US。有关支持的语言子类型列表，参见支持的语言环境。

Cookie 域

这是一个域列表，当在验证过程中向用户的浏览器设置了 Cookie 时，Cookie 标题中将返回该列表。如果列表为空，则不设置 Cookie 域。换句话说，Access Manager 会话 Cookie 将只会发送到 Access Manager 本身，而不发送到域中的其他服务器。

如果域中的其他服务器需要 SSO，该属性必须和 Cookie 域一同设置。如果在一个 Access Manager 上的不同域中有两个接口，则需要在该属性中对两个 Cookie 域进行设置。如果使用了负载平衡器，则 Cookie 域必须是负载平衡器的域，而不是负载平衡器背后的服务器。该字段的默认值是安装的 Access Manager 的域。

登录服务 URL

该字段用于指定登录页面的 URL。该属性的默认值为 /Service_DEPLOY_URI/UI/Login。

注销服务 URL

该字段用于指定注销页面的 URL。该属性的默认值为 /Service_DEPLOY_URI/UI/Logout。

可用的语言环境

该属性用于存储为平台配置的所有可用的语言环境。以一个允许用户选择自身语言环境的应用程序为例。该应用程序将从平台配置文件中获取该属性，并向用户提供语言环境列表。用户可选择一个语言环境，应用程序随即会在用户条目 preferredLocale 中设置该语言环境。

客户机字符集

该属性用于为位于平台级别上的不同客户机指定字符集。它包含一个客户机类型及其相应字符集的列表。有关更多信息，参见创建新的字符集。
