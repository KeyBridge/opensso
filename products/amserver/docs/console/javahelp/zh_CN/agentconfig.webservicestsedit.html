安全令牌服务

安全令牌服务是提供颁发和管理安全令牌的 Web 服务。即它常常以加密的设置发出安全性语句或声明，即使这些安全性语句或声明并非必要。这些语句以能够直接验证的证据或从信任授权机构获得的安全令牌为基础。要断言信任，服务可能会通过提供 STS 发布的一个或一组安全令牌来证明其权利，或者它可以发布带有自己信任语句的安全令牌（请注意，对于某些安全令牌格式，这可能只是重新发布或联合签名）。这构成了信任代理的基础。

常规

以下“常规”属性定义安全令牌服务的基本属性：

组

可通过“组”机制来定义相似代理类型的集合。组必须在将特定代理纳入集合之前进行定义。

密码

定义安全令牌服务属性的密码。

密码确认

确认密码。

状态

定义安全令牌服务代理是否参与保护外部服务请求。默认情况下，该属性设置为激活，即参与保护。

通用标识符

列出用于唯一定义安全令牌服务代理的基本 LDAP 属性。

安全

以下属性定义安全令牌服务的安全属性：

安全机制

定义用于保护安全令牌服务请求的安全证书类型。可选择以下安全类型：

  • 匿名 — 匿名安全机制不包含安全证书。

  • LibertyDiscoverySecurity — 使用基于 Liberty 的安全令牌来保护请求。

  • SAML-HolderOfKey — 使用 SAML 1.1 声明类型 Holder-Of-Key 来保护请求。

  • SAML-SenderVouches — 使用 SAML 1.1 声明类型 Sender Vouches 来保护请求。

  • SAML2–HolderOfKey — 使用 SAML 2.0 声明令牌类型 Holder-Of-Key 来保护请求。

  • SAML2–SenderVouches — 使用 SAML 2.0 声明令牌类型 Sender Vouches 来保护请求。

  • STSSecurity — 定义安全令牌服务代理从安全令牌服务获取安全令牌。

  • UserNameToken — 使用用户名令牌来保护安全令牌服务请求。

  • UserNameToken-Plain — 使用带有明文密码的用户名令牌来保护安全令牌服务请求。

  • X509Token — 使用 X509 证书来保护安全令牌服务代理请求。

STS 配置

当安全令牌服务代理使用安全令牌服务 (Security Token service, STS) 时，启用该属性。该配置描述用于保护 STS 服务请求的 STS 代理配置文件列表。

保留消息中的安全头

启用后，该属性定义为安全令牌服务保留 SOAP 安全头，以进行进一步处理。

用户令牌的证书

该属性表示安全令牌服务代理用来生成用户名安全令牌的用户名/密码共享密钥。

签署和加密

以下属性定义安全令牌服务的签署和加密配置：

请求是否已签名

启用后，安全令牌服务代理使用给定的令牌类型签署请求。

请求标头是否已加密

启用后，将加密安全令牌服务代理的安全标头。

请求是否已加密

启用后，将启用安全令牌服务请求。

响应签名是否已验证

启用后，将验证安全令牌服务响应签名。

响应是否已解密

启用后，将解密安全令牌服务响应。

密钥库

以下属性配置用于证书存储和检索的密钥库：

Web 服务提供者的公钥别名

该属性定义用于加密 Web 服务请求的公共证书密钥别名。

私钥别名

该属性定义用于签署/解密 Web 服务响应的私有证书密钥别名。

密钥存储库用法

该配置定义是使用默认的密钥库，还是使用自定义的密钥库。必须为自定义密钥库定义以下值：

  • 密钥库位置

  • 密钥库密码

  • 密钥密码

端点

以下属性定义 Web 服务端点：

Web 服务端点

该属性定义 Web 服务客户机发送请求的 Web 服务端点。
