配置请求处理程序

RequestHandler 接口必须由每一个使用 SOAP 绑定服务且基于 Liberty 的身份服务在服务器端实现。请求处理程序列表属性存储有关实现请求处理程序的 Web 服务实现类的信息。请求处理程序列表显示的条目包含用竖线分隔的关键字/值对，格式如下：key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService。

  1. 单击“新建”以显示“新建请求处理程序”属性，或单击已配置的关键字值以修改现有属性。

  2. 基于以下信息，为属性提供值：

    关键字

    该必填参数是 SOAP 端点的 URI 路径的最后一部分。Access Manager 中的 SOAP 端点是 SOAP 接收方 servlet。SOAP 接收方的 URI 为 protocol ://host:port/ deploy-uri/Liberty/key。如果将 disco 定义为关键字，则相应搜索服务的 SOAP 端点 URI 路径应该是： protocol://host:port /amserver/Liberty/disco。在与 SOAP 接收方建立连接时，不同的服务客户机使用不同的关键字。

    类

    该必填参数为特定的身份服务指定请求处理程序实现类的名称。例如：class=com.example.identity.liberty.ws.disco.DiscoveryService。

    SOAP 操作

    该参数用于指定特定于 SOAP 的可选 URI。

  3. 单击“确定”完成“请求处理程序”的配置操作。

  4. 单击“SOAP 绑定服务”页面上的“保存”完成服务的配置操作。
