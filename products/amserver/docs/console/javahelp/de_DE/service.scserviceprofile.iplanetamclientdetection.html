Client-Erkennung

Ein erster Schritt beim Authentifizierungsprozess besteht in der Ermittlung des Clienttyps, von dem die HTTP(S)-Anforderung ausgeht. In Access Manager wird diese Funktion als Client-Erkennung bezeichnet. Die URL-Informationen dienen zum Abrufen der Client-Merkmale. Auf Grundlage dieser Merkmale werden die entsprechenden Authentifizierungsseiten zurückgegeben. Wenn beispielsweise ein Netscape-Browser zum Anfordern einer Webseite verwendet wird, wird von Access Manager 7.1 eine HTML-Anmeldeseite angezeigt. Nach der Validierung des Benutzers wird der Client-Typ (Netscape-Browser) dem Sitzungs-Token hinzugefügt. Bei den Attributen des Client-Erkennungsdienstes handelt es sich um globale Attribute.

Client-Typen

Zur Feststellung des Client-Typs muss Access Manager die kennzeichnenden Merkmale des Clients erkennen. Diese Merkmale identifizieren die Eigenschaften aller unterstützten Typen mithilfe von Client-Daten. Mit diesem Attribut können Sie die Client-Daten über die Client-Manager-Schnittstelle ändern. Zum Öffnen des Client-Managers klicken Sie auf “Bearbeiten". Access Manager bietet die folgenden Client-Typen:

  • HDML

  • HTML

  • JHTML

  • VoiceX

  • WML

  • XHTML

  • cHTML

  • iHTML

Beschreibungen zu diesen Client-Typen finden Sie im http://docs.sun.com/app/docs/coll/1483.1.

Client-Manager

Diese Schnittstelle listet die Basis-Clients, Stile und zugehörigen Eigenschaften auf und ermöglicht das Hinzufügen und Konfigurieren von Geräten. Die Basis-Client-Typen sind ganz oben im Client-Manager aufgelistet. Diese Client-Typen enthalten die Standardeigenschaften, die von allen Geräten übernommen werden können, die dem jeweiligen Client-Typ zugeordnet sind.

Client-Typ

Der Client-Manager fasst alle verfügbaren Clients einschließlich des Basis-Client-Typs in der Liste “Client-Typ" zusammen. Wenn Sie die Eigenschaften eines Clients ändern möchten, klicken Sie auf dessen Gerätenamen. Die Eigenschaften werden daraufhin im Client-Editor angezeigt. Zur Bearbeitung der Eigenschaften wählen Sie die folgenden Klassifizierungen aus der Pulldown-Liste aus:

Hardware-Plattform

Umfasst Eigenschaften der Gerätehardware wie Auflösung, unterstützte Zeichensätze usw.

Software-Plattform

Umfasst Eigenschaften der Gerätesoftware wie Anwendungsumgebung, Betriebssystem und installierte Software.

Netzwerkeigenschaften

Umfasst Eigenschaften, die die Netzwerkumgebung einschließlich der unterstützten Träger beschreiben.

BrowserUA

Umfasst Attribute des auf dem Gerät laufenden Browser-Benutzeragenten.

WapCharacteristics

Umfasst Eigenschaften der vom Gerät unterstützten WAP-Umgebung (Wireless Application Protocol).

PushCharacteristicNames

Umfasst Eigenschaften der vom Gerät unterstützten WAP-Umgebung.

Weitere Eigenschaften

Umfasst Eigenschaften der vom Gerät unterstützten WAP-Umgebung (Wireless Application Protocol).

Hinweis – Die Definitionen bestimmter Eigenschaften finden Sie im Open Mobile Alliance Ltd. (OMA) Wireless Application Protocol, Version 20-Oct-2001 unter http://www1.wapforum.org/tech/terms.asp?doc=WAP-248-UAProf-20011020-a.pdf.

Um auf dieses Dokument zugreifen zu können, müssen Sie sich möglicherweise zunächst bei WAP Forum™ registrieren. Informationen hierzu finden Sie unter http://www.wapforum.org/faqs/index.htm.

Standard-Client-Typ

Legt den Standard-Client-Typ fest, der aus der Liste des Attributs “Client-Typen" ausgewählt wird. Die Standardeinstellung ist “genericHTML".

Client-Erkennungsklasse

Legt die Client-Erkennungsklasse fest, von der alle Client-Erkennungsanforderungen verarbeitet werden. Der von diesem Attribut zurückgegebene Wert sollte mit einem der im Attribut “Client-Typen" aufgeführten Typen übereinstimmen. Die standardmäßige Client-Erkennungsklasse lautet com.sun.mobile.cdm.FEDIClientDetector. Access Manager enthält außerdem com.iplanet.services.cdm.ClientDetectionDefaultImpl .

Client-Erkennung

Aktiviert die Client-Erkennung. Wenn die Client-Erkennung aktiviert ist, wird jede Anforderung von der im Attribut Client-Erkennungsklasse angegebenen Klasse verarbeitet. Standardmäßig ist die Client-Erkennung aktiviert. Wenn dieses Attribut deaktiviert ist, geht Access Manager von einem Client des Typs “genericHTML" aus, auf den über einen HTML-Browser zugegriffen wird.
