So konfigurieren Sie einen Site-Bezeichner

Dieses Attribut legt die Sites auf dem Server fest, auf dem Access Manager installiert ist. Für den Host werden bereits während der Installation ein Standardwert und eine automatisch generierte Site-ID festgelegt (diese Werte werden aus AMConfig.properties abgerufen). Es sind mehrere Einträge möglich. Beispielsweise müssten Load Balancing oder mehrere Instanzen von Access Manager, die denselben Directory Server nutzen, jeweils definiert werden. Der Ausgangspunkt ist das Attribut “Site-Bezeichner" im SAML-Bildschirm der Verbindungsschnittstelle.

  1. Klicken Sie auf “Neu", um einen neuen Site-Bezeichner hinzuzufügen, oder klicken Sie auf den Namen eines bereits konfigurierten Site-Bezeichners, um dessen Profil zu ändern.

    Die Site-Bezeichner-Attribute werden angezeigt.

  2. Legen Sie die Werte der Site-Bezeichner-Attribute entsprechend der folgenden Informationen fest:

    Instanz-ID

    Der Wert dieser Eigenschaft lautet protocol ://host: port.

    Wenn Sie SAML für SSL konfigurieren (sowohl in der Quell- als auch in der Ziel-Site), müssen Sie hier als Protokoll https// angeben.

    Site-ID

    Die Site-ID ist ein für jede Site generierter Bezeichner (bei mehreren für den Lastenausgleich konfigurierten Servern ist dieser Wert allerdings identisch). Das Paket com.sun.identity.saml.common enthält eine Klasse, mit der dieser Bezeichner bei Bedarf manuell generiert werden kann. Geben Sie in die Befehlszeile folgenden Befehl ein:

    % java -classpath FM-classpath com.sun.identity.saml.common.SAMLSiteID
protocol://host:port

    Ausstellername

    Der Standardwert für diese Eigenschaft lautet host :port, es könnte sich jedoch um jeden beliebigen URI handeln.

  3. Klicken Sie auf “OK", um die Konfiguration des Site-Bezeichners abzuschließen.

  4. Klicken Sie auf der Seite “SAML-Profil" auf “Speichern", um die SAML-Konfiguration abzuschließen.
