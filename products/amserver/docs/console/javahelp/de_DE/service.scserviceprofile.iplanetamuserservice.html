Benutzer

Die Standard-Benutzereinstellungen sind über den Benutzerdienst vordefiniert. Dazu zählen Zeitzone, Ländereinstellung und DN-Startansicht. Die Benutzerdienstattribute sind dynamische Attribute.

  • Von Benutzer bevorzugte Sprache

  • Von Benutzer bevorzugte Zeitzone

  • Vererbte Ländereinstellungen

  • Admin-DN-Startansicht

  • Standard-Benutzerstatus

Von Benutzer bevorzugte Sprache

Dieses Feld gibt die vom Benutzer ausgewählte Sprache für die in der Access Manager-Konsole angezeigten Text an. Der Standardwert ist “en". Mit diesem Wert wird der Benutzersitzung ein Lokalisierungsschlüsselsatz zugeordnet, so dass der am Bildschirm angezeigten Text in der vom Benutzer bevorzugten Sprache angzeigt wird.

Von Benutzer bevorzugte Zeitzone

Dieses Feld gibt die Zeitzone an, in der sich der Benutzer befindet, wenn er auf die Access Manager-Konsole zugreift. Es gibt keinen Standardwert.

Vererbte Ländereinstellungen

Dieses Feld gibt die Ländereinstellung für den Benutzer an. Der Standardwert ist en_US. Eine Liste der Ländereinstellungen finden Sie unter Unterstützte Sprach- bzw. Ländereinstellungen.

Admin-DN-Startansicht

Wenn der Benutzer ein Access Manager-Administrator ist, gibt dieses Feld den Knoten an, der bei der Benutzeranmeldung als Startpunkt für die Anzeige in der Access Manager-Konsole verwendet wird. Es gibt keinen Standardwert. Es kann ein gültiger DN verwendet werden, auf den der Benutzer mindestens Lesezugriff haben muss.

Standard-Benutzerstatus

Diese Option gibt den Standardstatus für alle neu erstellten Benutzer an. Dieser Status wird durch den Status des Benutzereintrags ersetzt. Nur aktive Benutzer können durch Access Manager authentifiziert werden. Der Standardwert ist “Aktiv". Wählen Sie eine der folgenden Optionen aus dem Pulldown-Menü aus:

Aktiv

Der Benutzer kann durch Access Manager authentifiziert werden.

Inaktiv

Der Benutzer kann nicht durch Access Manager authentifiziert werden, das Benutzerprofil verbleibt jedoch im Verzeichnis.

Der individuelle Benutzerstatus wird in vier Schritten eingerichtet: Zunächst wird der Benutzerdienst registriert und der zu verwendende Wert ausgewählt. Anschließend wird dieser Wert auf eine Rolle bezogen und diese wiederum dem Benutzerprofil hinzugefügt.
