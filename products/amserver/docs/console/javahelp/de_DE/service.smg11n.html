Globalisierungseinstellungen

Der Dienst “Globalisierungseinstellungen" enthält globale Attribute, mit denen Sie Access Manager für verschiedene Ländereinstellungen und Zeichensätze konfigurieren können. Es existieren folgende Attribute:

  • Von den einzelnen Ländereinstellungen unterstützte Zeichensätze

  • Zeichensatz-Aliasnamen

  • Format für allgemeinen Namen automatisch generiert

Von den einzelnen Ländereinstellungen unterstützte Zeichensätze

Dieses Attribut listet die für die einzelnen Ländereinstellungen unterstützten Zeichensätze auf und gibt so die Zuordnung zwischen Ländereinstellung und Zeichensatz an. Das Attribut hat folgendes Format:

Um einen neuen unterstützten Zeichensatz hinzuzufügen, klicken Sie auf “Hinzufügen" und legen Sie folgende Parameter fest:

Ländereinstellung

Die neue Ländereinstellung, die hinzugefügt werden soll. Weitere Informationen finden Sie unter Unterstützte Sprach- bzw. Ländereinstellungen.

Unterstützte Zeichensätze

Geben Sie den für die angegebene Ländereinstellung unterstützten Zeichensatz ein. Zeichensätze werden durch ein Semikolon voneinander getrennt. Beispiel: charset=charset1;charset2;charset3;...;charsetn

Um einen vorhandenen unterstützten Zeichensatz zu bearbeiten, klicken Sie in der Tabelle mit den unterstützten Zeichensätzen auf den entsprechenden Namen. Klicken Sie auf “OK", wenn Sie fertig sind.

Zeichensatz-Aliasnamen

Dieses Attribut listet die Namen der Codesätze (die IANA-Namen zugeordnet sind) auf, die für das Senden der Antwort verwendet werden. Diese Codesatz-Namen müssen nicht mit den Java-Codesatz-Namen übereinstimmen. Eine Hash-Tabelle für das Zuordnen von Java-Zeichensätzen zu IANA-Zeichensätzen und umgekehrt steht derzeit zur Verfügung.

Um einen neuen Zeichensatz-Aliasnamen hinzuzufügen, klicken Sie auf die Schaltfläche “Hinzufügen" und legen Sie folgende Parameter fest:

MIME-Name

Der IANA-Zuordnungsname. Beispiel: Shift_JIS

Java-Name

Der Java-Zeichensatz, der dem IANA-Zeichensatz zugeordnet werden soll.

Um einen vorhandenen unterstützten Zeichensatz-Alias zu bearbeiten, klicken Sie in der Tabelle mit den unterstützten Zeichensätzen auf den entsprechenden Namen. Klicken Sie auf “OK", wenn Sie fertig sind.

Format für allgemeinen Namen automatisch generiert

Die Anzeigeoption ermöglicht es Ihnen, festzulegen, auf welche Weise die automatische Namensgenerierung entsprechend den Namensformaten für verschiedene Ländereinstellungen und Zeichensätze erfolgen soll. Die Standardsyntax lautet wie folgt (beachten Sie, dass in der Definition angegebene Kommata und/oder Leerzeichen im Namensformat angezeigt werden):

en_us = {givenname} {initials} {sn}

Wenn Sie beispielsweise ein neues Namesnformat für einen Benutzer (Benutzer Eins) mit einer UID (1111) für den chinesischen Zeichnsatz anzeigen möchten, legen Sie Folgendes fest:

zh = {sn}{givenname}({uid})

Es wird Folgendes angezeigt:

OneUser 11111
