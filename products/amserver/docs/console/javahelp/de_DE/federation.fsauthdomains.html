Authentifizierungsdomänen

Eine Authentifizierungsdomäne ist eine Verbindung aus mehreren Dienstanbietern und mindestens einem Identity-Anbieter, mit denen Principals Geschäftstransaktionen in einer sicheren und nahtlosen Umgebung ausführen können. Zum Erstellen und Auffüllen einer Authentifizierungsdomäne erstellen Sie zunächst eine Entity zur Verwaltung der Metadaten (Konfigurationsinformationen, die eine bestimmte Identitätsdienstarchitektur definieren), und zwar für jeden Anbieter, der Mitglied der Authentifizierungsdomäne sein soll. Dann konfigurieren und speichern Sie die Authentifizierungsdomäne. Abschließend bearbeiten Sie die Eigenschaften der Entity, um der Authentifizierungsdomäne eine Entity (einen konfigurierten Anbieter) hinzuzufügen.

Hinweis – Weitere Informationen zu Entities finden Sie unter Entities: Anbieter und Partner.

Alle konfigurierten Authentifizierungsdomänen sind unter der entsprechenden Überschrift aufgelistet. In Verbindung mit Authentifizierungsdomänen müssen die folgenden Aufgaben ausgeführt werden:

  • So erstellen Sie eine neue Authentifizierungsdomäne

  • So ändern Sie das Profil einer Authentifizierungsdomäne

  • So fügen Sie einer Authentifizierungsdomäne Anbieter hinzu

  • So löschen Sie das Profil einer Authentifizierungsdomäne

Hinweis – Die Mitglieder einer Authentifizierungsdomäne müssen im Vorfeld eine Circle of Trust-Betriebsvereinbarung basierend auf den Liberty Trust Model Guidelines getroffen haben.
