Typen von Federated Access Manager Data Store

Im Auswahlbildschirm für Federated Access Manager-Data Stores können Sie folgende Data Store-Typen erstellen:

Federated Access Manager-Repository-Plugin

Diese Art von Data Store befindet sich in einer Sun Java System Directory Server-Instanz und enthält den Federated Access Manager-Informationsbaum. Diese Art von Data Store nutzt die Directory Server-Funktionen, die nicht Teil der LDAP Version 3-Spezifikation sind, beispielsweise Rollen und Class of Service (CoS), und ist mit älteren Versionen von Federated Access Manager kompatibel.

Attributdefinitionen finden Sie unter Federated Access Manager-Repository-Attribute.

Active Directory

Diese Art von Data Store verwendet die LDAP Version 3-Spezifikation, um Identitätsdaten in eine Instanz von Microsoft Active Directory zu schreiben.

Attributdefinitionen finden Sie unter Active Directory-Attribute.

Repository für Flat Files

Dieses Repository ermöglicht Ihnen das Speichern von Daten und Identitäten in einer Verzeichnisinformationsbaum-Flat-Struktur (Directory Information Tree, DIT) in der lokalen Installationsinstanz von Federated Access Manager, ohne einen separaten Data Store erstellen zu müssen. Im Allgemeinen kommt dies beim Testen oder Prüfen von Konzeptbereitstellungen zum Einsatz.

Attributdefinitionen finden Sie unter Attribute des Flat File-Repositorys.

Generisches LDAPv3

Bei diesem Data Store-Typ können Identitätsdaten in jede mit LDAPv3 kompatible Datenbank geschrieben werden. Wenn die von Ihnen verwendete LDAPv3-Datenbank die dauerhafte Suche nicht unterstützt, können Sie die Caching-Funktion nicht verwenden.

Attributdefinitionen finden Sie unter Generische LDAPv3-Attribute.

Sun Directory Server mit Federated Access Manager-Schema

Diese Art von Data Store befindet sich in einer Sun Java System Directory Server-Instanz und enthält den Federated Access Manager-Informationsbaum. Er unterscheidet sich dahin gehend vom Federated Access Manager-Repository-Plugin, dass durch eine größere Anzahl an Konfigurationsattributen die genauere Anpassung des Data Stores ermöglicht wird.

Attributdefinitionen finden Sie unter Sun Directory Server mit Federated Access Manager-Schemaattributen.
