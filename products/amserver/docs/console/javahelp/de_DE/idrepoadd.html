Access Manager-Data Store-Typen

Im Auswahlbildschirm für Access Manager-Data Stores können folgende Data Store-Typen erstellt werden:

Access Manager-Repository-Plugin

Diese Art von Data Store befindet sich in einer Sun Java System Directory Server-Instanz und enthält den Access Manager-Informationsbaum. Diese Art von Data Store nutzt die Directory Server-Funktionen, die nicht Teil der LDAP Version 3-Spezifikation sind, beispielsweise Rollen und Class of Service (CoS), und ist mit älteren Versionen von Access Manager kompatibel.

Attributdefinitionen finden Sie unter Access Manager-Repository-Attribute.

Active Directory

Diese Art von Data Store verwendet die LDAP Version 3-Spezifikation, um Identitätsdaten in eine Instanz von Microsoft Active Directory zu schreiben.

Attributdefinitionen finden Sie unter Active Directory-Attribute.

Repository für Flat Files

Dieses Repository ermöglicht Ihnen das Speichern von Daten und Identitäten in einer Verzeichnisinformationsbaum-Flat-Struktur (Direcotry Information Tree, DIT) in der lokalen Installationsinstanz von Access Manager, ohne einen separaten Data Store erstellen zu müssen. Im Allgemeinen kommt dies beim Testen oder Prüfen von Konzeptbereitstellungen zum Einsatz.

Attributdefinitionen finden Sie unter Attribute des Flat File-Repositorys.

Generisches LDAPv3

Bei diesem Data Store-Typ können Identitätsdaten in jede mit LDAPv3 kompatible Datenbank geschrieben werden. Wenn die von Ihnen verwendete LDAPv3-Datenbank die dauerhafte Suche nicht unterstützt, können Sie die Caching-Funktion nicht verwenden.

Attributdefinitionen finden Sie unter Generische LDAPv3-Attribute.

Sun Directory Server mit Access Manager-Schema

Diese Art von Data Store befindet sich in einer Sun Java System Directory Server-Instanz und enthält den Access Manager-Informationsbaum. Er unterscheidet sich dahin gehend vom Access Manager-Repository-Plugin, dass durch eine größere Anzahl an Konfigurationsattributen die genauere Anpassung des Data Stores ermöglicht wird.

Attributdefinitionen finden Sie unter Sun Directory Server mit Access Manager-Schemaattributen.
