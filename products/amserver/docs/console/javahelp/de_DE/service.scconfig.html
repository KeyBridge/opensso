Konfigurationsattribute

Die Konfigurationsseite enthält alle Attribute für die Konfiguration der Federated Access Manager-Standarddienste. Die Attribute eines Federated Access Manager-Dienstes sind in die folgenden Typen eingeteilt (einige Dienste können auch mehrere Typen umfassen):

Global – In der gesamten Federated Access Manager-Konfiguration angewendet. Globale Attribute dienen allein der Anpassung der Access Manager-Anwendung und können daher nicht auf einzelne Benutzer, Rollen oder Bereiche angewendet werden.

Bereich – Bereichsattribute werden ausschließlich Bereichen zugewiesen. Diese Attribute haben keine Objektklassen. Die unter den Authentifizierungsdiensten aufgelisteten Attribute gelten als Bereichsattribute, da die Authentifizierung eher auf Bereichsebene, als auf Teilbaum- oder Benutzerebene stattfindet.

Dynamisch – Einer in Federated Access Manager konfigurierten Rolle bzw. einem konfigurierten Bereich zugewiesen. Wird die Rolle einem Benutzer zugewiesen oder ein Benutzer in einem Bereich erstellt, kann das dynamische Attribut zu einer Eigenschaft des Benutzers werden.

Benutzer – Jedem Benutzer direkt zugewiesen. Sie werden nicht von einer Rolle oder einem Bereich übernommen und unterscheiden sich in der Regel von Benutzer zu Benutzer.

Folgende Konfigurationseigenschaften können geändert werden:

  • Authentifizierung

  • Konsoleneigenschaften

  • Globale Eigenschaften

  • Systemeigenschaften
