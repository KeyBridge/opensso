Konfigurationsattribute

Auf der Seite "Konfiguration" können Administratoren Attributwerte der von Federated Access Manager angebotenen Dienste verwalten. Die Attribute, die einen Federated Access Manager-Dienst umfassen, werden als einer der folgenden Typen klassifiziert:

Global – In der gesamten Federated Access Manager-Konfiguration angewendet. Globale Attribute dienen allein der Anpassung von Federated Access Manage und können daher nicht auf einzelne Benutzer, Rollen oder Bereiche angewendet werden.

Bereich – Bereichsattribute werden ausschließlich Bereichen zugewiesen. Diese Attribute haben keine Objektklassen. So gelten beispielsweise die in den Authentifizierungsdiensten aufgelisteten Attribute als Bereichsattribute, da die Authentifizierung eher auf Bereichsebene, als auf Unterstruktur- oder Benutzerebene stattfindet.

Dynamisch – Bezieht sich auf eine in Federated Access Manager konfigurierten Rolle bzw. einem dort konfigurierten Bereich. Wird die Rolle einem Benutzer zugewiesen oder ein Benutzer in einem Bereich erstellt, kann das dynamische Attribut zu einer Eigenschaft des Benutzers werden.

Benutzer – Bezieht sich unmittelbar auf die einzelnen Benutzer. Sie werden nicht von einer Rolle oder einem Bereich übernommen und unterscheiden sich in der Regel von Benutzer zu Benutzer.

Folgende Konfigurationsattribute können geändert werden:

  • Authentifizierung

  • Konsoleneigenschaften

  • Globale Eigenschaften

  • Systemeigenschaften
