Configuration d'un gestionnaire de demandes

L'interface RequestHandler doit être implémentée côté serveur par chaque service d'identité Liberty utilisant Service de liaison SOAP. L'attribut Liste de gestionnaires de demandes stocke les informations sur les classes d'implémentation des services Web qui implémentent le gestionnaire de demandes. La liste des gestionnaires de demandes affiche les entrées qui contiennent des paires clé/valeur séparées par un tube comme dans : key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService .

  1. Cliquez sur Nouveau pour afficher les attributs Nouveau gestionnaire de demandes ou cliquez sur une valeur de clé configurée pour modifier les attributs existants.

  2. Fournissez les valeurs pour les attributs en vous basant sur les informations suivantes :

    key

    Ce paramètre obligatoire correspond à la dernière partie du chemin d'URI vers une extrémité SOAP. Le point d'arrivée SOAP dans Access Manager est le servlet récepteur SOAP. L'URI pointant le récepteur SOAP est protocol ://host:port/ deploy-uri/Liberty/key. Si vous définissez disco comme clé, le chemin d'URI vers l'extrémité SOAP du Service de découverte correspondant est : protocol://host:port /amserver/Liberty/disco. Chaque client de service utilise une clé différente pour se connecter au récepteur SOAP.

    class

    Ce paramètre obligatoire définit le nom de la classe d'implémentation du gestionnaire de demandes pour ce service d'identités particulier. Par exemple : class=com.exemple.identité.liberty.ws.disco.DiscoveryService.

    Action SOAP

    Ce paramètre définit un URI spécifique à SOAP facultatif.

  3. Cliquez sur OK pour terminer la configuration du gestionnaire de demandes.

  4. Cliquez sur Enregistrer dans la page Service de liaison SOAP pour terminer la configuration du service.
