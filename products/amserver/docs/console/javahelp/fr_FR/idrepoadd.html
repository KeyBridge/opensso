Types de magasins de données Federated Access Manager

Dans l'écran de sélection de magasin de données Federated Access Manager, vous pouvez créer les types de magasins de données suivants :

Plug-in de référentiel Federated Access Manager

Ce type de magasin de données se trouve dans une instance Sun Java System Directory Server et contient l'arborescence d'informations Federated Access Manager. Il utilise les fonctions Directory Server ne faisant pas partie de la spécification LDAP version 3, comme les rôles et la classe de service, et est compatible avec les versions antérieures de Federated Access Manager.

Pour des définitions d'attributs, reportez-vous à Attributs de référentiel Federated Access Manager.

Active Directory

Ce type de magasin de données utilise la spécification LDAP version 3 pour écrire des données d'identité dans une instance de Microsoft Active Directory.

Pour des définitions d'attributs, reportez-vous à Attributs Active Directory.

Référentiel fichiers plats

Ce référentiel vous permet de stocker des données et des identités dans une structure DIT simple sur l'instance d'installation locale de Federated Access Manager sans créer de magasin de données distinct. Il est généralement utilisé pour tester ou approuver des déploiements types.

Pour des définitions d'attributs, reportez-vous à Attributs du référentiel fichiers plats.

LDAPv3 générique

Ce type de magasin de données vous permet d'écrire des données d'identité dans une quelconque base de données compatible LDAPv3. Si la base de données LDAPv3 utilisée ne prend pas en charge la recherche persistante, vous ne pourrez pas utiliser la fonction de mise en cache.

Pour des définitions d'attributs, reportez-vous à Attributs LDAPv3 générique.

Sun Directory Server avec Federated Access Manager Schema

Ce type de magasin de données se trouve dans une instance Sun Java System Directory Server et contient l'arborescence d'informations Federated Access Manager. Il est différent du plug-in du référentiel de Federated Access Manager en ce qu'un plus grand nombre d'attributs de configuration vous permet de mieux personnaliser le magasin de données.

Pour des définitions d'attributs, reportez-vous à Attributs Sun Directory Server avec Federated Access Manager Schema.
