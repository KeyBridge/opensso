Configuration d'une offre de ressource pour l'amorçage

Une seule offre de ressources est autorisée pour l'amorçage ; par défaut, cette offre contient des informations concernant le Service de découverte. Si une offre de ressource est déjà définie, vous pouvez modifier les attributs en cliquant sur son nom. Vous pouvez également cocher la case en regard du nom et supprimer l'offre de ressource existante. Pour configurer une nouvelle offre de ressources, cliquez sur Nouveau.

  1. Cliquez sur Nouveau pour afficher les attributs du nouveau mappeur ou cliquez sur le nom d'un mappeur configuré pour le modifier.

  2. Saisissez en option une description de l'offre de ressource.

  3. Fournissez les valeurs pour les attributs d'instance de service en vous basant sur les informations suivantes :

    Type de service

    Cet attribut accepte comme valeur un URI définissant le type de service implémenté par l'offre de services. Par exemple, urn:liberty:disco:2003-08.

    Remarque – Il est recommandé que cet URI soit identique à l'URI targetNamespace de la description WSDL abstraite du service.

    ID fournisseur

    Cet attribut accepte la valeur d'un URI du fournisseur de l'instance du service. Par exemple, http://daiquiri.sun.com:80/amserver/Liberty/disco.

    Description du service

    Cet attribut définit un service Web en cours d'exécution au niveau d'une extrémité de protocole distincte. Les informations sur les instances de service doivent être communiquées dans différents contextes. Par exemple, le Service de découverte défini dans cette spécification est un service d'identité qui fournit une liste d'offres de ressource (dont chacune inclut une description d'instance de service). Reportez-vous à Configuration d'une description de service.

  4. Fournissez les valeurs pour les attributs d'options d'offre de ressource en vous basant sur les informations suivantes :

    Options

    Cochez cette case si ce service ne fournit aucune option pour l'offre de ressource. Les options fournissent au demandeur potentiel des indications sur la disponibilité de certaines données ou opérations avec une offre de ressource particulière. Par exemple, une option indiquant que des informations de contact sont disponibles peut être fournie.

    Liste des options

    Cet attribut contient une liste d'options pour l'instance de service. Cette option est définie comme un URI. L'ensemble des URI possibles est généralement standardisé par le type de service.

  5. Activez des directives basées sur les informations suivantes :

    Réf. d'ID de description pour : GenerateBearerToken

    Toutes les Directives prises en charge peuvent contenir une référence descriptive. Si l'attribut Réf. d'ID n'est pas activé pour une directive, la directive est appliquée à tous les mécanismes d'authentification fournis dans l'offre de ressources. Si l'attribut Réf. d'ID est activé pour une directive, il DOIT contenir la liste des ID de référence qui font référence au mécanisme d'authentification auquel la directive est associée. Si l'attribut est présent, la directive DOIT être appliquée uniquement aux descriptions référencées dans la liste des ID de référence. Ceci peut être utile si certaines directives sont incompatibles avec certains mécanismes de sécurité.
