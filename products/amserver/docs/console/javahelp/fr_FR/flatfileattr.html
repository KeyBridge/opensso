Attributs du référentiel fichiers plats

Les attributs suivants permettent de configurer un référentiel fichiers plats :

Nom de classe de plug-in du référentiel de fichiers

Cet attribut spécifie le fichier de classe Java fournissant l'implémentation de fichiers plats. Cet attribut ne doit pas être modifié.

Répertoire référentiel de fichiers

Définit le répertoire de base dans lequel les identités et leurs attributs sont stockés.

Cache

Si activé (par défaut), les identités et leurs attributs sont mis en cache. Les requêtes ultérieures n'accèdent pas au système de fichiers.

Heure de mise à jour du cache

Lorsque la mise en cache est activée, cet attribut détermine l'intervalle (en minutes) après lequel les entrées du cache sont vérifiées à la recherche de modifications apportées au système de fichiers. Le mécanisme de vérification est basé sur des horodatages.

Classes d'objets utilisateur de fichier

Définit les classes d'objets ajoutées automatiquement aux utilisateurs à leur création.

Attribut du mot de passe

Fournit le nom d'attribut contenant le mot de passe utilisé pour l'authentification. Cet attribut permet d'authentifier l'utilisateur lorsque le module d'authentification Magasin de données est activé.

Attribut du statut

Fournit le nom d'attribut stockant le statut de l'identité. Les valeurs de l'attribut du statut sont active et inactive. Il est utilisé pendant l'authentification de l'identité. Si une identité est inactive, l'utilisateur n'est pas authentifié.

Attributs hachés

Fournit une liste d'attributs dont les valeurs sont hachées et stockées dans les fichiers. Une fois hachées, les valeurs d'origine ne peuvent pas être obtenues. Seules les valeurs hachées sont récupérées. Il est utilisé pour garantir la confidentialité selon laquelle certains attributs ne doivent pas être stockés de manière permanente mais dans un objectif de vérification. Un attribut de mot de passe d'identité est un exemple d'un tel type d'attribut.

Attributs chiffrés

Fournit une liste d'attributs dont les valeurs sont chiffrées et stockées dans les fichiers. Bien qu'elles soient chiffrées et stockées, l'appel d'API du référentiel d'identité peut renvoyer les valeurs déchiffrées d'origine. Ceci empêche les utilisateurs d'accéder directement au système de fichiers et de consulter des attributs critiques.
