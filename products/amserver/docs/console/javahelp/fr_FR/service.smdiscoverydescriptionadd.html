Configuration d'une description de service

L'attribut Description du service définit un service Web en cours d'exécution au niveau d'une extrémité de protocole distincte. Il est défini lorsque vous configurez les Offres de ressources pour l'amorçage. Les informations sur les instances de service doivent être communiquées dans différents contextes. Par exemple, le Service de découverte défini est un service d'identité qui fournit une liste des offres de ressources (dont chacune inclut une description d'instance de service).

  1. Cliquez sur Ajouter une description pour afficher les attributs de la nouvelle description de service ou cliquez sur Modifier pour modifier une description existante.

  2. Fournissez les valeurs pour les attributs en vous basant sur les informations suivantes :

    ID de mécanisme de sécurité

    Cet attribut définit l'emplacement où sont ajoutées les méthodes d'authentification prises en charge par le Service de découverte. Ces mécanismes de sécurité font référence à la manière dont un consommateur de service Web s'authentifie auprès du fournisseur de services Web ou fournit la sécurité au niveau des messages. Par défaut, toutes les méthodes disponibles prises en charge par l'instance du service sont sélectionnées. Si une méthode d'authentification n'est pas sélectionnée et que le consommateur de service Web envoie une demande à l'aide de cette méthode, la demande est rejetée. Reportez-vous à Mécanismes d'authentification pris en charge.

    URL de point d'arrivée

    La valeur de cet attribut est l'URL de point d'arrivée SOAP sur HTTP. Le plan URI doit être HTTP ou HTTPS comme dans https://soap.profile-provider.com/soap .

    Action SOAP

    La valeur de cet attribut en option équivaut à l'attribut wsdlsoap:soapAction de l'élément wsdlsoap:operation dans la description WSDL du service.

  3. Cliquez sur OK pour terminer la configuration du service.

  4. Cliquez sur Enregistrer dans la page du Service de découverte pour terminer la configuration du service.
