Liste de gestionnaires de demandes

Service de liaison SOAP fournit l'interface RequestHandler pour traiter le messages de demande et renvoyer une réponse. Cette interface doit être implémentée côté serveur par chaque service d'identité basé sur Liberty utilisant Service de liaison SOAP. L'attribut Liste de gestionnaires de demandes stocke les informations sur les classes d'implémentation des services Web qui implémentent le gestionnaire de demandes. La liste des gestionnaires de demandes affiche les entrées qui contiennent des paires clé/valeur séparées par un tube comme dans : key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService . Ces paramètres sont expliqués dans le tableau suivant.

Paramètre

En-tête

key

Le paramètre obligatoire key est la dernière partie du chemin de l'URI vers une extrémité SOAP. L'extrémité SOAP dans Access Manager est le servlet récepteur SOAP. L'URI du récepteur SOAP est : protocol://host:port /deploy-uri/Liberty/key Si vous définissez disco comme paramètre, le chemin de l'URI vers l'extrémité SOAP du Service de découverte correspondant est : protocol://host:port /amserver/Liberty/disco. Des clients de services différents utilisent des clés différentes pour la connexion au récepteur SOAP.

class

Le paramètre obligatoire class spécifie le nom de la classe d'implémentation RequestHandler du service d'identité. Par exemple : class=com.exemple.identité.liberty.ws.disco.DiscoveryService.

Pour ajouter un nouveau gestionnaire de demandes, reportez-vous à Configuration d'un gestionnaire de demandes.
