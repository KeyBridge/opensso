Introduction

Sun Java System Federated Access Manager empêche les accès non autorisés aux applications de service Web et aux contenus Web. Federated Access Manager intègre des services d'authentification et d'autorisation, des agents de stratégie et la fédération des entités afin de fournir une solution complète pour la protection de vos ressources réseau.

De nombreuses entreprises accordent les accès aux informations sur la base de l'application. Par exemple, un employé peut avoir à définir un nom d'utilisateur et un mot de passe pour accéder au site Web d'administration de la mutuelle d'entreprise. Le même employé peut avoir un nom d'utilisateur et un mot de passe différents pour accéder aux formulaires en ligne du service Comptabilité. Au sein de la même entreprise, un client définit un nom d'utilisateur et un mot de passe pour accéder à la partie publique du site Web de la société. Pour chaque site Web ou service, un administrateur doit convertir l'entrée de l'utilisateur de l'entreprise en un format de données identifiable par le service. Chaque service ajouté à l'entreprise doit être complété et géré séparément.

Federated Access Manager réduit les coûts administratifs et élimine les informations utilisateur redondantes associées aux solutions par application. Federated Access Manager permet à un administrateur d'assigner des règles ou des stratégies spécifiques qui régissent les informations ou services auxquels chaque utilisateur peut accéder. Les agents de stratégie sont déployés sur les serveurs d'applications ou les serveurs Web afin de traiter les demandes HTTP et de faire appliquer les stratégies actives. Conjointement, les informations de l'utilisateur et les stratégies d'accès associées constituent l'identité de l'entreprise de l'utilisateur. Federated Access Manager permet à un utilisateur d'accéder à de nombreuses ressources de l'entreprise sous une seule identité.
