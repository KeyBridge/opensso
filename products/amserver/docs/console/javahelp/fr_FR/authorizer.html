Agent d'autorisation

Avant de traiter une demande, Service de profil personnel Liberty vérifie l'autorisation du WSC ayant émis cette demande. Le contrôle d'autorisation peut être effectué à deux niveaux :

  1. L'entité demandeuse est-elle autorisée à accéder aux informations de profil de la ressource requise ?

  2. La ressource requise est-elle publiée pour le demandeur ?

L'autorisation est transmise via un plug-in à Service de profil personnel Liberty, une implémentation de la classe com.sun.identity.liberty.ws.interfaces.Authorizer. Bien qu'une nouvelle implémentation puisse être développée, Access Manager fournit la classe par défaut com.sun.identity.liberty.ws.idpp.plugin.IDPPAuthorizer. Ce plug-in définit quatre valeurs d'action de stratégie pour les opérations de requête et de modification :

  • Autoriser

  • Refuser

  • Interaction pour l'acceptation

  • Interaction pour la valeur

Les valeurs de ressource pour les règles sont similaires aux expressions x-path définies par Service de profil personnel Liberty.

/PP/CommonName/AnalyzedName/FN 	Demande 	Interaction pour l'acceptation
/PP/CommonName/*                      	Modification 	Interaction pour la valeur
/PP/InformalName                        	Demande 	Refus
