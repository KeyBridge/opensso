Fournisseur de services Web

Le profil d'agent du fournisseur de services Web décrit la configuration utilisée pour sécuriser les demandes de service Web d'un fournisseur de services Web. Les noms des services Web doivent être uniques à chaque fournisseur de services Web.

D'ordre général

Les attributs d'ordre général suivants définissent les propriétés du fournisseur de services Web de base :

Groupe

Le mécanisme de groupe permet de définir un ensemble de types d'agents similaires. Vous devez définir le groupe avant d'inclure l'agent spécifique dans un ensemble.

Mot de passe

Définit le mot de passe de l'attribut du fournisseur de services Web.

Confirmation de mot de passe

Confirmez le mot de passe.

Statut

Détermine si l'agent du fournisseur de services Web participe ou non à la sécurisation des demandes de service sortantes. Par défaut, cet attribut est défini sur Active, ce qui signifie que l'agent participe à la sécurisation.

Identificateur universel

Répertorie les propriétés LDAP de base qui définissent de manière unique l'agent du fournisseur de services Web.

Sécurité

Les attributs suivants définissent les attributs de sécurité du fournisseur de services Web :

Mécanisme de sécurité

Définit le type de référence de sécurité utilisé pour sécuriser la demande du fournisseur de services Web. Vous pouvez effectuer un choix parmi les types de sécurité suivants :

  • Anonyme — Le mécanisme de sécurité anonyme ne contient aucune référence de sécurité.

  • LibertyBearerToken

  • LibertySAMLToken

  • LibertyX509Token — Utilise le certificat X509 défini par Liberty pour sécuriser la demande du fournisseur Web.

  • SAML-HolderOfKey — Utilise le type d'assertion SAML 1.1 Holder-Of-Key pour sécuriser la demande.

  • SAML-SenderVouches — Utilise le type d'assertion SAML 1.1 Sender Vouches pour sécuriser la demande.

  • SAML2–HolderOfKey — Utilise le type de jeton d'assertion SAML 2.0 Holder-Of-Key pour sécuriser la demande.

  • SAML2–SenderVouches — Utilise le type de jeton d'assertion SAML 2.0 Sender Vouches pour sécuriser la demande.

  • UserNameToken — Utilise un jeton de nom d'utilisateur pour sécuriser les demandes du client du fournisseur Web.

  • UserNameToken-Plain — Utilise un jeton de nom d'utilisateur avec un mot de passe en texte clair pour sécuriser les demandes du fournisseur Web.

  • X509Token — Utilise le certificat X509 pour sécuriser les demandes du fournisseur Web.

Chaîne d'authentification

Définition –

Préserver les en-têtes de sécurité dans les messages

Lorsque cet attribut est activé, il indique que les en-têtes de sécurité SOAP sont conservés pour le fournisseur de services Web en vue d'un traitement approfondi.

URN de type de service Liberty

L'URN (Universal Resource Name) décrit un type de service Liberty que le fournisseur de services Web utilise pour les recherches de services.

Références du jeton d'utilisateur

L'attribut représente les secrets partagés nom d'utilisateur/mot de passe que le fournisseur de services Web utilise pour générer un jeton de sécurité de nom d'utilisateur.

Type de clé privée

Définition –

Configuration SAML

Les attributs suivants configurent le langage SAML (Security Assertion Markup Language) du fournisseur de services Web :

Mappage d'attribut SAML

Cette configuration représente un attribut SAML devant être généré par le service de jeton de sécurité pendant la création de l'assertion. Le même mappage est utilisé pour être intégré au jeton SSOToken par le fournisseur de services Web.

Plug-in de mappeur NameID SAML

Définit la classe de plug-in de mapper NameID utilisée pour le mappage de compte SAML.

Espace de noms d'attributs SAML

Définit l'espace de noms utilisé pour générer les attributs SAML.

Inclure appartenances

Lorsque cet attribut est activé, il indique que l'appartenance à l'identité principale doit être incluse en tant qu'attribut SAML.

Signature et chiffrement

Les attributs suivants définissent la configuration de signature et de chiffrement pour la sécurité du fournisseur Web :

La réponse est signée

Lorsque cet attribut est activé, le fournisseur de services Web signe la réponse avec un type de jeton donné.

La réponse est chiffrée

Lorsque cet attribut est activé, la réponse du service Web est chiffrée.

La signature de demande est vérifiée

Lorsque cet attribut est activé, la signature de demande du service Web est vérifiée.

L'en-tête de demande est chiffré

Lorsque cet attribut est activé, l'en-tête de sécurité du client du service Web est chiffrée.

La demande est déchiffrée

Lorsque cet attribut est activé, la demande du fournisseur de services Web est déchiffrée.

Keystore

Les attributs suivants configurent le keystore à utiliser pour le stockage et l'extraction du certificat :

Alias de clé publique du client du service Web

Cet attribut définit l'alias de clé de certificat publique utilisé pour chiffrer la réponse du service Web.

Alias de clé privée

Cet attribut définit l'alias de clé de certificat privée utilisé pour signer/déchiffrer la demande du service Web.

Utilisation du keystore

Cette configuration spécifie d'utiliser le keystore par défaut ou un keystore personnalisé. Les valeurs suivantes doivent être définies pour un keystore personnalisé :

  • Emplacement du keystore

  • Mot de passe du keystore

  • Mot de passe de clé

Points d'arrivée

Les attributs suivants définissent les points d'arrivée du service Web :

Point d'arrivée de service Web

Cet attribut définit un point d'arrivée de service Web auquel le client du service Web fait une demande.
