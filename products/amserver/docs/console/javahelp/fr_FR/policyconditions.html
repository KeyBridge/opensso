Conditions

Une condition vous permet de définir des contraintes dans la stratégie. Par exemple, si vous définissez une stratégie pour une application de paiement des salaires, vous pouvez définir une condition sur cette action limitant l'accès à l'application à des heures spécifiques. Ou bien, vous pouvez définir une condition qui autorise cette action uniquement si la demande émane d'un ensemble d'adresses IP particulier ou du réseau interne d'une entreprise.

La condition peut en outre être utilisée pour configurer différentes stratégies sur différents URI du même domaine. Par exemple, http://org.example.com/hr/*jsp n'est accessible que via org.example.net de 9h à 17h. Ceci est possible à l'aide d'une condition IP et d'heure. Et en spécifiant la ressource de règle sous la forme http://org.example.com/hr/*.jsp , la stratégie s'applique à tous les JSP sous http://org.example.com/hr , y compris ceux dans les sous-répertoires.

Remarque – Les termes orientation, règle, ressource, objet, condition, action et valeur correspondent aux éléments Orientation, Règle, Nom de ressource, Objet, Condition, Attribut et Valeur dans le fichier policy.dtd.
