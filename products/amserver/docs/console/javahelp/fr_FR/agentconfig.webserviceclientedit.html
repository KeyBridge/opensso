Attributs du client de service Web

Le profil d'agent du client du service Web décrit la configuration utilisée pour sécuriser les demandes de service Web sortantes d'un client de service Web. Les noms des services Web doivent être uniques à chaque client de service Web.

D'ordre général

Les attributs d'ordre général suivants définissent les propriétés du client de service Web de base :

Groupe

Le mécanisme de groupe permet de définir un ensemble de types d'agents similaires. Vous devez définir le groupe avant d'inclure l'agent spécifique dans un ensemble.

Mot de passe

Définit le mot de passe de l'attribut du client du service Web.

Confirmation de mot de passe

Confirmez le mot de passe.

Statut

Détermine si l'agent du client du service Web participe ou non à la sécurisation des demandes de service sortantes. Par défaut, cet attribut est défini sur Active, ce qui signifie que l'agent participe à la sécurisation.

Identificateur universel

Répertorie les propriétés LDAP de base qui définissent de manière unique l'agent du client du service Web.

Sécurité

Les attributs suivants définissent les attributs de sécurité du client du service Web :

Mécanisme de sécurité

Définit le type de référence de sécurité utilisé pour sécuriser la demande du client du service Web. Vous pouvez effectuer un choix parmi les types de sécurité suivants :

  • Anonyme — Le mécanisme de sécurité anonyme ne contient aucune référence de sécurité.

  • LibertyDiscoverySecurity — Utilise les jetons de sécurité Liberty pour sécuriser la demande.

  • SAML-HolderOfKey — Utilise le type d'assertion SAML 1.1 Holder-Of-Key pour sécuriser la demande.

  • SAML-SenderVouches — Utilise le type d'assertion SAML 1.1 Sender Vouches pour sécuriser la demande.

  • SAML2–HolderOfKey — Utilise le type de jeton d'assertion SAML 2.0 Holder-Of-Key pour sécuriser la demande.

  • SAML2–SenderVouches — Utilise le type de jeton d'assertion SAML 2.0 Sender Vouches pour sécuriser la demande.

  • STSSecurity — Spécifie que le client du service Web obtient le jeton de sécurité auprès du service de jeton de sécurité.

  • UserNameToken — Utilise un jeton de nom d'utilisateur pour sécuriser les demandes du client du service Web.

  • UserNameToken-Plain — Utilise un jeton de nom d'utilisateur avec un mot de passe en texte clair pour sécuriser les demandes du service Web.

  • X509Token — Utilise le certificat X509 pour sécuriser les demandes du service Web.

Configuration STS

Cet attribut est activé lorsque le client du service Web utilise le service de jeton de sécurité (STS, Security Token Service). Cette configuration décrit une liste de profils d'agent STS utilisés pour sécuriser les demandes au service STS.

Configuration de la détection

Cet attribut est activé lorsque le client du service Web est activé pour la sécurité du service de découverte. Cette configuration décrit une liste de profils d'agent de découverte utilisés pour sécuriser les demandes au service de découverte.

Authentification utilisateur requise

Lorsque cet attribut est activé, il spécifie que la page protégée du client des services nécessite que l'utilisateur soit authentifié pour obtenir l'accès.

Préserver les en-têtes de sécurité dans les messages

Lorsque cet attribut est activé, il indique que les en-têtes de sécurité SOAP sont conservés pour le client du service Web en vue d'un traitement approfondi.

URN de type de service Liberty

L'URN (Universal Resource Name) décrit un type de service Liberty que le client du service Web utilise pour les recherches de services.

Références du jeton d'utilisateur

L'attribut représente les secrets partagés nom d'utilisateur/mot de passe que le client du service Web utilise pour générer un jeton de sécurité de nom d'utilisateur.

Signature et chiffrement

Les attributs suivants définissent la configuration de signature et de chiffrement pour la sécurité du service Web :

La demande est signée

Lorsque cet attribut est activé, le client du service Web signe la demande avec un type de jeton donné.

L'en-tête de demande est chiffré

Lorsque cet attribut est activé, l'en-tête de sécurité du client du service Web est chiffré.

La demande est chiffrée

Lorsque cet attribut est activé, la demande du client du service Web est activée.

La signature de réponse est vérifiée

Lorsque cet attribut est activé, la signature de réponse du service Web est vérifiée.

La réponse est déchiffrée

Lorsque cet attribut est activé, la réponse du service Web est déchiffrée.

Keystore

Les attributs suivants configurent le keystore à utiliser pour le stockage et l'extraction du certificat :

Alias de clé publique du fournisseur de services Web

Cet attribut définit l'alias de clé de certificat publique utilisé pour chiffrer la demande du service Web.

Alias de clé privée

Cet attribut définit l'alias de clé de certificat privée utilisé pour signer/déchiffrer la réponse du service Web.

Utilisation du keystore

Cette configuration spécifie d'utiliser le keystore par défaut ou un keystore personnalisé. Les valeurs suivantes doivent être définies pour un keystore personnalisé :

  • Emplacement du keystore

  • Mot de passe du keystore

  • Mot de passe de clé

Points d'arrivée

Les attributs suivants définissent les points d'arrivée du service Web :

Point d'arrivée de service Web

Cet attribut définit un point d'arrivée de service Web auquel le client du service Web fait une demande.
