Utilisateur

Les préférences utilisateur par défaut sont définies grâce au service d'utilisateur. Ces préférences incluent le fuseau horaire, la langue et la vue initiale du DN. Les attributs de service utilisateur sont des attributs dynamiques.

  • Langue de préférence de l'utilisateur

  • Fuseau horaire favori de l'utilisateur

  • Langue héritée

  • Vue initiale de l'administrateur

  • Statut utilisateur par défaut

Langue de préférence de l'utilisateur

Ce champ spécifie le choix de l'utilisateur concernant la langue d'affichage du texte sur la console Access Manager. La valeur par défaut est en. Cette valeur mappe un ensemble de clés de localisation sur la session utilisateur de manière à ce que le texte apparaisse à l'écran dans une langue adaptée à l'utilisateur.

Fuseau horaire favori de l'utilisateur

Ce champ spécifie le fuseau horaire dans lequel l'utilisateur accède à la console d'Access Manager. Il n'y a pas de valeur par défaut.

Langue héritée

Ce champ spécifie la langue de l'utilisateur. La valeur par défaut est en_US (anglais américain). Pour consulter la liste des langues, reportez-vous à Environnements linguistiques pris en charge.

Vue initiale de l'administrateur

Si l'utilisateur est un administrateur d'Access Manager, ce champ spécifie le nœud servant de point de départ affiché sur la console d'Access Manager lorsque cet utilisateur se connecte. Il n'y a pas de valeur par défaut. Il est possible d'utiliser un DN valide pour lequel l'utilisateur dispose au minimum de l'accès en lecture.

Statut utilisateur par défaut

Cette option indique le statut par défaut de tout utilisateur récemment créé. Ce statut est remplacé par le statut de l'entrée utilisateur. Seuls les utilisateurs actifs peuvent s'authentifier par l'intermédiaire d'Access Manager. La valeur par défaut est Actif. Vous pouvez sélectionner l'une ou l'autre de ces options dans le menu déroulant :

Actif

L'utilisateur peut s'authentifier par l'intermédiaire d'Access Manager.

Inactif

L'utilisateur ne peut pas s'authentifier par l'intermédiaire d'Access Manager, mais le profil utilisateur reste stocké dans le répertoire.

Pour définir le statut de l'utilisateur individuel par l'enregistrement du service utilisateur, vous devez choisir la valeur, l'appliquer à un rôle et l'ajouter au profil de cet utilisateur.
