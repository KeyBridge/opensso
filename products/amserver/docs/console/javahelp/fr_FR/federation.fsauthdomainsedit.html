Modification d'un profil de domaine d'authentification

Suivez cette procédure pour modifier les attributs globaux d'un domaine d'authentification existant ou pour lui ajouter des fournisseurs. Le point de départ est Domaines d'authentification sous l'interface Fédération.

  1. Cliquez sur le nom d'un domaine d'authentification configuré afin de modifier son profil ou de lui ajouter des fournisseurs.

    La page Éditer le domaine d'authentification s'affiche.

  2. Tapez les nouvelles valeurs ou modifiez celles présentes dans les attributs généraux du domaine d'authentification :

    Nom

    La valeur statique de cet attribut est le nom fourni lors de la création du domaine d'authentification.

    Description

    Cet attribut est une description du domaine d'authentification. Si nécessaire, vous pouvez modifier la description déjà saisie.

    URL du service écriture

    Cet attribut indique l'emplacement du service qui écrit le cookie du domaine commun. L'URL a le format http://common-domain-host:port/common/writer .

    URL du service lecture

    Cet attribut indique l'emplacement du service qui lit le cookie du domaine commun. L'URL a le format http://common-domain-host:port/common/transfer .

    État

    Le statut par défaut est Actif. La sélection de Inactif désactive les communications au sein du domaine d'authentification.

  3. Cliquez sur Ajouter pour ajouter des fournisseurs au domaine d'authentification.

    La page de sélection d'un partenaire de confiance s'affiche.

  4. Choisissez un ou plusieurs fournisseurs disponibles et cliquez sur la flèche Ajouter pour les sélectionner.

    La liste fournie contient les noms des entités créées et renseignées avec des fournisseurs. Pour plus d'informations, voir Ajout de fournisseurs à un domaine d'authentification.

  5. Cliquez sur OK pour enregistrer les fournisseurs du domaine d'authentification.

    Vous revenez à l'écran Profil du domaine d'authentification.

  6. Cliquez sur Enregistrer pour terminer l'opération.
