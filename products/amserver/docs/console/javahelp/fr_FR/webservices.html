Services Web

Le module Services Web est l'implémentation d'Access Manager des spécifications Liberty Alliance Project Identity Web Services Framework (ID-WSF). Basée sur la spécification Liberty Identity Federation Framework (ID-FF), ID-WSF fournit les spécifications nécessaires pour implémenter les services Web qui récupèrent, mettent à jour ou exécutent une action sur les données d'identité dans un environnement de réseau fédéré. Ces spécifications présentent les composants techniques nécessaires pour mettre en oeuvre les services Web, par exemple un service de calendrier, un service de portefeuille ou un service d'alerte.

Remarque – Les données d'identité incluent, entre autres, les attributs qui définissent le prénom, le nom, l'adresse personnelle, l'adresse professionnelle et une personne à contacter en cas d'urgence.

Les services Web constituent la base de l'informatique distribuée via Internet. Un consommateur de service Web (WSC) localise un service Web et appelle les opérations proposées par ce service en transmettant une demande. Un fournisseur de services Web (WSP) est une application qui implémente le service Web en fonction de la demande ; celle-ci peut se trouver sur la même machine virtuelle Java™ que le WSC ou à plusieurs milliers de kilomètres. Lorsqu'un WSC a besoin de récupérer des attributs d'identité auprès d'un WSP, il doit d'abord contacter un service de découverte afin de localiser l'emplacement spécifique des attributs. Une fois ces informations retournées, le WSC peut alors contacter le WSP (par exemple, un service de profil personnel) afin de récupérer les attributs nécessaires.

Le module Services Web de la console d'Access Manager fournit une interface permettant de configurer les éléments suivants :

  • Service de profil personnel Liberty

  • Service de découverte

  • Service de liaison SOAP

  • Service Web d'authentification
