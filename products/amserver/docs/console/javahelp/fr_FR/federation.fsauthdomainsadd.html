Création d'un domaine d'authentification

Suivez cette procédure pour créer un domaine d'authentification. Le point de départ est Domaines d'authentification sous l'interface Fédération.

  1. Cliquez sur Nouveau pour afficher les attributs du domaine d'authentification.

    La page de profil du nouveau domaine d'authentification s'affiche.

  2. Tapez le nom du domaine d'authentification.

  3. Tapez la description du domaine d'authentification dans le champ Description.

  4. Tapez la valeur de l'URL du service écriture.

    L'URL du service écriture indique l'emplacement du servlet qui écrit le cookie du domaine commun. Utilisez le format http://common-domain-host :port/common/writer.

  5. Tapez la valeur de l'URL du service lecture.

    L'URL du service lecture indique l'emplacement du servlet qui lit le cookie du domaine commun. Utilisez le format http://common-domain-host :port/common/transfer.

  6. Choisissez Actif ou Inactif.

    Le statut par défaut est Actif. Le choix de Inactif désactive les communications au sein du domaine d'authentification.

  7. Cliquez sur OK pour terminer la configuration.

    Le nouveau domaine d'authentification apparaît dans l'écran Domaines d'authentification.
