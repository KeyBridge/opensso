Service de jeton de sécurité

Un service de jeton de sécurité est un service Web qui émet et gère les jetons de sécurité. Cela signifie que les instructions ou les réclamations de sécurité figurent souvent dans des ensembles chiffrés, bien que cela ne soit pas obligatoire. Ces instructions reposent sur la preuve qu'il peut procéder à une vérification directe ou sur des jetons de sécurité provenant d'autorités de confiance. Pour établir la confiance, un service peut démontrer son droit à revendiquer un ensemble de réclamations en fournissant un jeton de sécurité ou un ensemble de jetons de sécurité émis par un service de jeton de sécurité, ou il peut émettre un jeton de sécurité avec sa propre instruction de confiance (pour certains formats de jeton de sécurité, il peut simplement s'agir d'une ré-émission ou d'une co-signature). Ceci forme la base de l'établissement de confiance.

D'ordre général

Les attributs d'ordre général suivants définissent les propriétés du service de jeton de sécurité de base :

Groupe

Le mécanisme de groupe permet de définir un ensemble de types d'agents similaires. Vous devez définir le groupe avant d'inclure l'agent spécifique dans un ensemble.

Mot de passe

Définit le mot de passe de l'attribut du service de jeton de sécurité.

Confirmation de mot de passe

Confirmez le mot de passe.

Statut

Détermine si l'agent du service de jeton de sécurité participe ou non à la sécurisation des demandes de service sortantes. Par défaut, cet attribut est défini sur Active, ce qui signifie que l'agent participe à la sécurisation.

Identificateur universel

Répertorie les propriétés LDAP de base qui définissent de manière unique l'agent du service de jeton de sécurité.

Sécurité

Les attributs suivants définissent les attributs de sécurité du service de jeton de sécurité :

Mécanisme de sécurité

Définit le type de référence de sécurité utilisé pour sécuriser la demande du service de jeton de sécurité. Vous pouvez effectuer un choix parmi les types de sécurité suivants :

  • Anonyme — Le mécanisme de sécurité anonyme ne contient aucune référence de sécurité.

  • LibertyDiscoverySecurity — Utilise les jetons de sécurité Liberty pour sécuriser la demande.

  • SAML-HolderOfKey — Utilise le type d'assertion SAML 1.1 Holder-Of-Key pour sécuriser la demande.

  • SAML-SenderVouches — Utilise le type d'assertion SAML 1.1 Sender Vouches pour sécuriser la demande.

  • SAML2–HolderOfKey — Utilise le type de jeton d'assertion SAML 2.0 Holder-Of-Key pour sécuriser la demande.

  • SAML2–SenderVouches — Utilise le type de jeton d'assertion SAML 2.0 Sender Vouches pour sécuriser la demande.

  • STSSecurity — Spécifie que l'agent du service de jeton de sécurité obtient le jeton de sécurité auprès du service de jeton de sécurité.

  • UserNameToken — Utilise un jeton de nom d'utilisateur pour sécuriser les demandes du service de jeton de sécurité.

  • UserNameToken-Plain — Utilise un jeton de nom d'utilisateur avec un mot de passe en texte clair pour sécuriser les demandes du service de jeton de sécurité.

  • X509Token — Utilise le certificat X509 pour sécuriser les demandes de l'agent du service de jeton de sécurité.

Configuration STS

Cet attribut est activé lorsque l'agent du service de jeton de sécurité utilise le service de jeton de sécurité (STS, Security Token Service). Cette configuration décrit une liste de profils d'agent STS utilisés pour sécuriser les demandes au service STS.

Préserver les en-têtes de sécurité dans les messages

Lorsque cet attribut est activé, il indique que les en-têtes de sécurité SOAP sont conservés pour l'agent du service de jeton de sécurité en vue d'un traitement approfondi.

Références du jeton d'utilisateur

L'attribut représente les secrets partagés nom d'utilisateur/mot de passe que l'agent du service de jeton de sécurité utilise pour générer un jeton de sécurité de nom d'utilisateur.

Signature et chiffrement

Les attributs suivants définissent la configuration de signature et de chiffrement pour le service de jeton de sécurité :

La demande est signée

Lorsque cet attribut est activé, l'agent du service de jeton de sécurité signe la demande avec un type de jeton donné.

L'en-tête de demande est chiffré

Lorsque cet attribut est activé, l'en-tête de sécurité de l'agent du service de jeton de sécurité est chiffré.

La demande est chiffrée

Lorsque cet attribut est activé, la demande du service de jeton de sécurité est activée.

La signature de réponse est vérifiée

Lorsque cet attribut est activé, la signature de réponse du service de jeton de sécurité est vérifiée.

La réponse est déchiffrée

Lorsque cet attribut est activé, la réponse du service de jeton de sécurité est déchiffrée.

Keystore

Les attributs suivants configurent le keystore à utiliser pour le stockage et l'extraction du certificat :

Alias de clé publique du fournisseur de services Web

Cet attribut définit l'alias de clé de certificat publique utilisé pour chiffrer la demande du service Web.

Alias de clé privée

Cet attribut définit l'alias de clé de certificat privée utilisé pour signer/déchiffrer la réponse du service Web.

Utilisation du keystore

Cette configuration spécifie d'utiliser le keystore par défaut ou un keystore personnalisé. Les valeurs suivantes doivent être définies pour un keystore personnalisé :

  • Emplacement de keystore

  • Mot de passe de keystore

  • Mot de passe de clé

Points d'arrivée

Les attributs suivants définissent les points d'arrivée du service Web :

Point d'arrivée de service Web

Cet attribut définit un point d'arrivée de service Web auquel le client du service Web fait une demande.
