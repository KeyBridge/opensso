Détection de client

L'étape initiale du processus d'authentification consiste à identifier le type du client qui émet la demande HTTP(S). Cette fonction Access Manager est appelée détection de client. Les informations d'URL sont utilisées pour récupérer les caractéristiques du client. Sur la base de ces caractéristiques, les pages d'authentification appropriées sont retournées. Par exemple, lorsqu'un navigateur Netscape est utilisé pour demander une page Web, Access Manager 7.1 affiche une page de connexion HTML. Une fois que l'utilisateur est validé, le type de client (navigateur Netscape) est ajouté au jeton de session. Les attributs définis dans le service Détection de client sont des attributs globaux.

Types de client

Pour pouvoir détecter les types de client, Access Manager doit reconnaître leurs caractéristiques d'identification. Ces caractéristiques identifient les propriétés de tous les types pris en charge dans le formulaire des données client. Cet attribut vous permet de modifier les données client par l'intermédiaire de l'interface Gestionnaire de clients. Pour accéder au Gestionnaire de clients, cliquez sur le lien Modifier. Par défaut, Access Manager contient les types de client suivants :

  • HDML

  • HTML

  • JHTML

  • VoiceX

  • WML

  • XHTML

  • cHTML

  • iHTML

Pour consulter la description de ces types de clients, reportez-vous au http://docs.sun.com/app/docs/coll/1483.1.

Gestionnaire de clients

Le gestionnaire de clients est l'interface qui répertorie les clients de base, leurs styles et leurs propriétés, et qui permet d'ajouter et de configurer des périphériques. Les types de client de base sont répertoriés en haut du gestionnaire de clients. Ces types de client contiennent les propriétés par défaut dont peuvent hériter tous les périphériques faisant partie de ce type de client.

Type de client

Profil de style : le Gestionnaire de clients regroupe tous les clients disponibles, y compris le client de base, dans la liste des types de client. Vous pouvez modifier les propriétés de chaque client en cliquant sur le nom du périphérique. Les propriétés sont ensuite affichées dans la fenêtre Éditeur du client. Pour modifier les propriétés, sélectionnez les classifications suivantes dans la liste déroulante :

Plate-forme matérielle

Contient les propriétés concernant le matériel du périphérique, par exemple la taille de l'écran, les jeux de caractères pris en charge, etc.

Plate-forme logicielle

Contient les propriétés de l'environnement d'application du périphérique, du système d'exploitation et des logiciels installés.

Caractéristiques du réseau

Contient les propriétés décrivant l'environnement réseau, notamment les services support pris en charge.

BrowserUA

Contient les attributs liés à l'agent utilisateur du navigateur exécuté sur le périphérique.

WapCharacteristics

Contient les propriétés de l'environnement WAP (Wireless Application Protocol) pris en charge par le périphérique.

PushCharacteristicNames

Contient les propriétés de l'environnement WAP pris en charge par le périphérique.

Autres propriétés

Contient les propriétés de l'environnement WAP (Wireless Application Protocol) pris en charge par le périphérique.

Remarque – Pour des définitions de propriétés spécifiques, reportez-vous au document Open Mobile Alliance Ltd. (OMA) Wireless Application Protocol, Version 20-Oct-2001 à l'adresse http://www1.wapforum.org/tech/terms.asp?doc=WAP-248-UAProf-20011020-a.pdf.

Pour accéder à ce document, vous devez vous enregistrer dans le WAP Forum™. Pour plus d'informations, consultez le site http://www.wapforum.org/faqs/index.htm.

Type de client par défaut

Cet attribut définit le type de client par défaut extrait de la liste des types de client dans l'attribut Types de client. La valeur par défaut est genericHTML.

Classe de détection de client

Cet attribut définit la classe de détection de client pour laquelle toutes les demandes de détection de client sont acheminées. La chaîne retournée par cet attribut doit correspondre à l'un des types de client répertoriés dans l'attribut Types de client. La classe de détection de client par défaut est com.sun.mobile.cdm.FEDIClientDetector. Access Manager contient également la classe com.iplanet.services.cdm.ClientDetectionDefaultImpl .

Détection de client

Active la détection de client. Si la détection de client est activée (valeur par défaut), toutes les demandes sont acheminées par l'intermédiaire de la classe définie dans l'attribut Classe de détection de client. Par défaut, la fonction de détection de client est activée. Si cet attribut n'est pas sélectionné, Access Manager suppose que le client est genericHTML et qu'il est accessible à partir d'un navigateur HTML.
