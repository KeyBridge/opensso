To Add a New Hosted Identity Provider to a Provider Entity

A hosted provider is hosted on the same server as Access Manager. Configuring the New Hosted Identity Provider attributes entails adding metadata concerning the identity provider to the entity profile. The starting point for this procedure is To Add an Identity Provider to a Provider Entity.

Note - Additional attributes are configurable but not visible from this initial configuration page. See To Modify Hosted Identity Provider Attributes in a Provider Entity.

  1. Provide information for the Common Attributes.

    Common Attributes contain values that generally define the identity provider itself.

    Description

    The value of this attribute is the description provided when you created the entity. You may modify the description originally entered or add one if none exists.

    Protocol Support Enumeration

    Choose the Liberty Alliance Project Liberty Identity Federation Framework (Liberty ID-FF) release supported by this entity.

    • urn:liberty:iff:2003-08 refers to the Liberty ID-FF version 1.2.

    • urn:liberty:iff:2002-12 refers to the Liberty ID-FF version 1.1.

  2. Provide information for the Communication URLs attributes.

    Communication URLs attributes contain locations to which providers can send transmissions earmarked for this identity provider.

    Single Sign-On Service URL

    Type a URL to which service providers can send single sign-on and federation requests.

  3. Provide information for the Communication Profiles attributes.

    A Liberty Alliance Project profile defines the combination of a message's content and the mechanism used to transport it. The following Communication Profiles attributes define the profiles used by the identity provider for the corresponding functionality.

    Single Sign-on/Federation

    Select the profile for this identity provider to use when sending responses to authentication or federation requests.

    • Browser Post (specifies a browser-based HTTP POST protocol)

    • Browser Artifact (specifies a non-browser SOAP-based protocol)

    • LECP (specifies a Liberty-enabled Client Proxy)

  4. Select any of the available circle of trust to assign to the provider.

    A provider can belong to one or more circle of trusts. However, a provider without a specified circle of trust can not participate in communications based on the Liberty Alliance Project specifications. If no circle of trusts have been created, you may define this attribute later.

  5. Provide information for the Federated Access Manager Configuration attributes.

    Access Manager Configuration attributes define information regarding this hosted instance of Federated Access Manager.

    Provider URL

    Type the URL of the provider hosted locally.

    Provider Alias

    The Provider Alias is a name used to identify the hosted provider within an entity profile and define whether it is a service provider or an identity provider configuration. The value must be unique and able to be represented in a URL. An example is sunAlias.

    Authentication Type

    Select what type of provider should be used for authentication requests from a provider hosted locally.

    • Local specifies that the provider hosted locally should contact a local identity provider upon receiving an authentication request (essentially, itself).

    • Remote specifies that the provider hosted locally would contact a remote identity provider upon receiving an authentication request.

    Default Authentication Context

    Select the authentication context class (method of authentication) to be used by this identity provider if the information is not received as part of a service provider request. This value also specifies the authentication context class to be used when an unknown user tries to access a service provider's protected resource and the service provider sends a request for authentication. The options are:

    • Password

    • Mobile Digital ID

    • Smartcard

    • Smartcard-PKI

    • MobileUnregistered

    • Software-PKI

    • Previous-Session

    • Mobile Contract

    • Time-Sync-Token

    • Password-ProtectedTransport

    Realm

    Type a value which points to the realm in which this provider is configured. For example, /sp.

    Liberty Version URI

    Type a URI specifying the version of the Liberty Alliance Project specification being used. The default value is http://projectliberty.org/specs/v1 .

    Name Identifier Implementation

    This field defines the class used by a service provider to participate in name registration. Name registration is a profile by which service providers specify a principal’s name identifier that an identity provider will use when communicating to the service provider. The value is com.sun.identity.federation.services.util.FSNameIdentifierImpl .

    Home Page URL

    Type the URL of the home page for the provider hosted locally.

    Single Sign-on Failure Redirect URL

    Type the URL to which a principal will be redirected if single sign-on has failed.

  6. Provide information for the SAML Attributes.

    SAML Attributes define general values regarding how the identity provider will send SAML assertions.

    Assertion Interval

    Type the interval of time (in seconds) that an assertion issued by the identity provider will remain valid. A principal will remain authenticated until the assertion interval expires.

    Cleanup Interval

    Type the interval of time (in seconds) before assertions stored in the identity provider will be cleared.

    Artifact Timeout

    Type the interval of time (in seconds) to specify when assertion artifacts will time out.

    Assertion Limit

    Type a number to define how many assertions an identity provider can issue, or how many assertions that can be stored.

  7. Click OK to add the provider to the entity.
  8. Continue configuring the entity by selecting another option from the View menu or click Save to complete the configuration.
