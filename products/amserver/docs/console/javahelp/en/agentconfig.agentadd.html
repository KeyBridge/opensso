To Create a New Agent

Creating a new agent is also referred to as creating a new agent profile. You can perform this task in the Federated Access Manager Administration Console. This task applies to web agents, J2EE agents, and 2.2 agents. However, some of the individual steps do not apply to the 2.2 agents.

  1. Click the Access Control tab.
  2. Click the name of the realm to which the agent will belong.
  3. Click the Agents tab.
  4. If necessary, select the tab for the appropriate agent type.
    • Web Agents:

      The Web tab is for web agents and is the default selection. No further selection is necessary.

    • J2EE Agents:

      The J2EE tab is for J2EE agents. If applicable, click this tab.

    • 2.2 Agents

      The 2.2 Agents tab is for agents in the Policy Agent 2.2 software set. If applicable, click this tab.

  5. In the Agent section, click New.
  6. In the Name field, enter the name for the new agent profile.
  7. Enter and confirm the Password.

    Caution – This password must be the same password that you enter in the agent profile password file that you specify when you run the agentadmin program to install the agent.

  8. Select the applicable configuration: Local or Centralized (Does not apply to 2.2 agents).

    When local configuration is selected, the properties related to this agent cannot be edited from the Console. In such a scenario, the agent retrieves configuration information from the local files, FAMAgentBootstrap.properties and FAMAgentConfiguration.properites, in the agent installation directory. For an agent configured locally, change property values by editing the FAMAgentConfiguration.properites file directly.

  9. In the Server URL field, enter the Federated Access Manager server URL (Does not apply to 2.2 agents).

    For example:

    http://famHost.example.com:8080/fam
  10. In the Agent URL field, enter the URL for the agent application, agentapp (Does not apply to 2.2 agents).

    For example:

    Web Agents

    http://agentHost.example.com:8090

    J2EE Agents

    http://agentHost.example.com:8090/agentapp

  11. Click Create.

    The Console creates the agent profile and displays the agent (WebAgent, J2EEAgent, or 2.2_Agent) page again, with a link to the new agent profile.

    To do additional configuration for the agent profile, click this link to display the Edit agent page.
