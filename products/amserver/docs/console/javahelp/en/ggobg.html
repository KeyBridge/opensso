To Create a New Group

If desired, perform this task in the Federated Access Manager Administration Console. This task applies to web agent groups and J2EE agent groups.

Create a group if you want agents to inherit specific properties from the group. J2EE agents can inherit properties from a J2EE agent group while Web agents can inherit properties from a web agent group.

  1. Click Agents under the Configuration tab.
  2. If necessary, select the tab for the appropriate agent type.
    • Web Agents:

      The Web tab is for web agents and is the default selection. No further selection is necessary.

    • J2EE Agents:

      The J2EE tab is for J2EE agents. If applicable, click this tab.

  3. In the Group section, click New.

    In the Name field, enter the name for the new group name.

  4. In the Server URL field, enter the Federated Access Manager server URL.
  5. Click Create.

    The console creates the agent group and displays the agent (WebAgent or J2EEAgent) page again with a link to the group.

    To do additional configuration of the group, click this link to display the Edit agent group page.

    The attributes you can set to configure a group are the same as they are for an individual agent except that the Group, Password, and Password Confirm attributes are not available at the group level.

    Note – Also, be aware that some group attributes already have variable values assigned that in most cases should not be changed. The following is one example of such a value:

    @AGENT_PROTO@://@AGENT_HOST@:@AGENT_PORT@/amagent
