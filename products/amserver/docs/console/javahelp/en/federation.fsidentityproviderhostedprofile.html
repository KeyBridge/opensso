To Modify Hosted Identity Provider Attributes in a Provider Entity

After creating an entity and adding an identity provider, you can edit the identity provider's profile. This might entail adding values to attributes that were not available to configure when originally creating the identity provider, or modifying configured data. The starting point is Entities under the Federation interface.

  1. Click the name of the configured provider entity to which the identity provider belongs.

    The entity's profile page is displayed.

  2. Select Identity Provider from the View menu.
  3. Modify values for the Common Attributes.

    Common Attributes contain values that generally define the identity provider itself.

    Provider Type

    The static value of this attribute defines whether this is a hosted or remote provider.

    Description

    The value of this attribute is the description provided when you created the entity. You may modify the description originally entered or add one if none exists.

    Protocol Support Enumeration

    Choose the Liberty Alliance Project Liberty Identity Federation Framework (Liberty ID-FF) release supported by this entity.

    • urn:liberty:iff:2003-08 refers to the Liberty ID-FF version 1.2.

    • urn:liberty:iff:2002-12 refers to the Liberty ID-FF version 1.1.

    Server Name Identifier Mapping Binding

    Name identifier mapping allows a service provider to obtain a name identifier for a principal that has federated in the namespace of a different service provider. Thus, the requesting provider can communicate with the other service provider about the principal without an identity federation for the principal between them. Type a URI that identifies the communication protocol used to obtain the identifier and click Add.

    Note - Currently, the Name Identifier Mapping profile in Access Manager only supports SOAP. If this attribute is used, its value must be http://projectliberty.org/profiles/nim-sp-http .

    Signing Key: Key Alias

    Certificates are stored in a JavaTM keystore file. Each certificate in the file is mapped to an alias used to fetch it. Type the key alias associated with the certificate used to sign requests and responses.

    Encryption Key: Key Alias

    Certificates are stored in a Java keystore file. Each certificate in the file is mapped to an alias used to fetch it. Type the key alias associated with the provider's security certificate.

    Encryption Key: Key Size

    Type the length for keys used by the web service consumer when interacting with another entity.

    Encryption Key: Encryption Method

    Choose the method of encryption. The choices include:

    • None

    • AES

    • DESede

    Name Identifier Encryption

    Select the check box to enable encryption of the name identifier.

  4. Modify values for the Communication URLs attributes.

    Communication URLs attributes contain locations to which providers can send transmissions earmarked for this identity provider.

    SOAP Endpoint

    Type a URL to the identity provider’s SOAP message receiver. This value communicates the location of the SOAP receiver in non browser communications.

    Single Sign-On Service URL

    Type a URL to which service providers can send single sign-on and federation requests.

    Single Logout Service

    Type a URL to which service providers can send logout requests. Single logout synchronizes the logout functionality across all sessions authenticated by this identity provider.

    Single Logout Return

    Type a URL to which the identity provider will redirect the principal after completing a logout.

    Federation Termination Service

    Type a URL to which a service provider will send federation termination requests.

    Federation Termination Return

    Type a URL to which the identity provider will redirect the principal after completing federation termination.

    Name Registration Service

    Type a URL to which a service provider will send requests to specify a name identifier to be used when communicating with this identity provider about a particular principal. This optional service can only be implemented after a federation session is established.

    Name Registration Return

    Type a URL to which the identity provider will redirect the principal after HTTP-redirect-based name registration has been completed.

  5. Modify values for the Communication Profiles attributes.

    A Liberty Alliance Project profile defines the combination of a message's content and the mechanism used to transport it. The following Communication Profiles attributes define the profiles used by the identity provider for each functionality.

    Federation Termination

    Select a profile for the identity provider to use in order to notify other providers of a principal’s federation termination.

    • HTTP Redirect

    • SOAP

    Single Logout

    Select a profile for the identity provider to use in order to notify other providers of a principal’s logout.

    • HTTP Redirect

    • HTTP Get

    • SOAP

    Name Registration

    Select a profile for other providers to use when contacting this identity provider in order to register a new name identifier for a principal.

    • HTTP Redirect

    • SOAP

    Single Sign-on/Federation

    Select the profile for this identity provider to use when sending responses to authentication or federation requests.

    • Browser Post (specifies a browser-based HTTP POST protocol)

    • Browser Artifact (specifies a non-browser SOAP-based protocol)

    • LECP (specifies a Liberty-enabled Client Proxy)

  6. Select any of the available circle of trusts to assign to the provider.

    A provider can belong to one or more circle of trusts. However, a provider without a specified circle of trust can not participate in communications based on the Liberty Alliance Project specifications. If no circle of trusts have been created, you may define this attribute later.

  7. Select the authentication context to be used if the identity provider does not receive the information as part of a service provider request.

    This attribute maps the authentication context classes defined in the Liberty Alliance Project specifications to authentication methods available at the identity provider. This value also specifies the authentication context used by the service provider when an unknown user tries to access a protected resource.

    Supported

    Select the check box next to the authentication context class if the identity provider supports it.

    Context Reference

    The authentication context classes defined in the Liberty Alliance Project specifications are:

    • Mobile Contract

    • Mobile Digital ID

    • MobileUnregistered

    • Password

    • Password-ProtectedTransport

    • Previous-Session

    • Smartcard

    • Smartcard-PKI

    • Software-PKI

    • Time-Sync-Token

    Key

    Choose the authentication type to which the context is mapped.

    Value

    Type the authentication option.

    Priority

    Choose a priority level for cases where there are multiple contexts.

  8. Choose from the available Trusted Providers and add to the entity.

    The list contains configured entities that have been populated with providers.

  9. Modify values for the Federated Access Manager Configuration attributes.

    Federated Access Manager Configuration attributes define information regarding this hosted instance of Federated Access Manager.

    Provider Alias

    The Provider Alias is a name used to identify the hosted provider within an entity profile and define whether it is a service provider or an identity provider configuration. The value must be unique and able to be represented in a URL. An example is sunAlias.

    Authentication Type

    Select what type of provider should be used for authentication requests from a provider hosted locally.

    • Local specifies that the provider hosted locally should contact a local identity provider upon receiving an authentication request (essentially, itself).

    • Remote specifies that the provider hosted locally would contact a remote identity provider upon receiving an authentication request.

    Default Authentication Context

    Select the authentication context class (method of authentication) to be used by this identity provider if the information is not received as part of a service provider request. This value also specifies the authentication context class to be used when an unknown user tries to access a service provider's protected resource and the service provider sends a request for authentication. The options are:

    • Password

    • Mobile Digital ID

    • Smartcard

    • Smartcard-PKI

    • MobileUnregistered

    • Software-PKI

    • Previous-Session

    • Mobile Contract

    • Time-Sync-Token

    • Password-ProtectedTransport

    Realm

    Type a value which points to the realm in which this provider is configured. For example, /sp.

    Liberty Version URI

    Type a URI specifying the version of the Liberty Alliance Project specification being used. The default value is http://projectliberty.org/specs/v1 .

    Name Identifier Implementation

    This field defines the class used by a service provider to participate in name registration. Name registration is a profile by which service providers specify a principal’s name identifier that an identity provider will use when communicating to the service provider. The value is com.sun.identity.federation.services.util.FSNameIdentifierImpl .

    Home Page URL

    Type the URL of the home page for the provider hosted locally.

    Single Sign-on Failure Redirect URL

    Type the URL to which a principal will be redirected if single sign-on has failed.

    Assertion Issuer

    Type the name of the host that issues the assertion. This value might be the load balancer's host name if Access Manager is behind one.

    Generate Discovery Bootstrapping Resource Offering

    Select the check box if you want a Discovery Service Resource Offering to be generated for bootstrapping purposes during the single sign on process based on the Liberty Alliance Project specifications.

    Auto Federation

    Auto-federation will automatically federate a user's disparate provider accounts based on a common attribute. Select the check box to enable auto federation.

    Auto Federation Common Attribute Name

    The value of this attribute will be used as the common attribute on which a user's disparate provider accounts will be automatically federated.

    Attribute Statement Plug-in

    Specify a pluggable class used for adding attribute statements to an assertion generated during the single sign on process based on the Liberty Alliance Project specifications.

    Identity Provider Attribute Mapping

    The values define the mappings used by the default attribute mapper plug-in. Mappings for this attribute should be configured in the format:

    SAML-attribute=local-attribute

    For example, EmailAddress=mail or Address=postaladdress ).

  10. Provide information for the SAML Attributes.

    SAML Attributes define general information regarding SAML assertions that are sent by the identity provider.

    Assertion Interval

    Type the interval of time (in seconds) that an assertion issued by the identity provider will remain valid. A principal will remain authenticated until the assertion interval expires.

    Cleanup Interval

    Type the interval of time (in seconds) before assertions stored in the identity provider will be cleared.

    Artifact Timeout

    Type the interval of time (in seconds) to specify the timeout for assertion artifacts.

    Assertion Limit

    Type a number to define how many assertions an identity provider can issue, or how many assertions that can be stored.

  11. Modify values for the Organization Profiles attributes.

    The Organizations Profile attributes provide basic information that may be required when interacting with a principal. These attributes are optional.

    Note - If the Names attribute contains a value, it is required to add values to the Display Names and URL attributes.

    Names

    Enter the legal name for the entity’s organization and click Add. The value is defined in the format locale| organization-name. For example, en| example.com.

    Display Names

    Enter a name for the entity’s organization that is suitable for display to a principal and click Add. The value is defined in the format locale|organization-display-name . For example, en|example.com.

    URL

    Enter the URL of the organization and click Add. The value is defined in the format locale|organization-URL . For example, en|http://www.example.com .

  12. To configure Contact Persons for the provider, click New Contact Person.

    This attribute is optional. For more information, see To Add a Contact Person to a Provider in a Provider Entity.

  13. Click Save to complete the modifications.
