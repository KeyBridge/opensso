Entities: Provider and Affiliate

An entity may be configured with metadata (configuration information that defines a particular identity service architecture) for an individual identity provider, an individual service provider, or one of each. Contrarily, an entity may be configured as an affiliation, a selected group of providers of either type. Both provider and affiliation entities can be configured using the Federation Access Manager Console.

Note – An entity can be created but it will not be available for assignment to a circle of trust until it has been populated with providers.

The two types of entities are described in the following table.

Table 1 Types of Entities

SAML2 Entity Provider

A provider entity holds the metadata for individual providers of either type. All identity providers and service providers must be configured within a provider entity before they can be associated with a circle of trust, or chosen to be included in an affiliate entity. Using the attributes provided in the Access Manager Console, one individual identity provider, one individual service provider, or one of each can be defined within a provider entity. A provider entity profile contains the following groupings of attributes:

  • General Attributes

  • Identity Provider

  • Service Provider

Affiliate Entity

A configured affiliation enables a user to federate with a group of associated sites. For example, when a service provider issues an authentication request for a user on behalf of an affiliation and authentication is secured, the user can achieve single sign-on with all members of the affiliation. An affiliation is formed without regard to the boundaries of a circle of trust.

An affiliate entity holds the metadata that defines the grouping of one or more provider entities that comprise the affiliation. It does not contain the configuration information for any providers, only the configuration information for the affiliation itself. An affiliate entity profile contains the following groupings of attributes:

  • General Attributes

  • Affiliate

Note – Members of an affiliation may invoke services either as a member of the affiliation, or individually.
