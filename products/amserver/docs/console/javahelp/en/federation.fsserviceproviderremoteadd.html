To Add a New Remote Service Provider to a Provider Entity

A remote provider is not hosted on the same server as Federated Access Manager. Configuring the New Remote Service Provider attributes entails adding metadata concerning the service provider to the entity profile. The starting point for this procedure is To Add a Service Provider to a Provider Entity.

Note – Additional attributes are configurable but not visible from this initial configuration page. .

  1. Provide information for the Common Attributes.

    Common Attributes contain values that generally define the identity provider itself.

    Description

    The value of this attribute is the description provided when you created the entity. You may modify the description originally entered or add one if none exists.

    Protocol Support Enumeration

    Choose the Liberty Alliance Project Liberty Identity Federation Framework (Liberty ID-FF) release supported by this entity.

    • urn:liberty:iff:2003-08 refers to the Liberty ID-FF version 1.2.

    • urn:liberty:iff:2002-12 refers to the Liberty ID-FF version 1.1.

  2. Select any of the available circle of trusts to assign to the provider.

    A provider can belong to one or more circle of trusts. However, a provider without a specified circle of trust can not participate in communications based on the Liberty Alliance Project specifications. If no circle of trusts have been created, you may define this attribute later.

  3. Provide information for the Service Provider attributes.

    Service Provider attributes define general information regarding the service provider.

    Assertion Consumer URL

    Type a URL to the end point which will receive and process all SAML assertions sent to the service provider.

    Assertion Consumer Service URL ID

    If the value of the Protocol Support Enumeration common attribute is urn:liberty:iff:2003-08, type the required ID.

    Set Assertion Consumer Service URL as Default

    Select the check box to use the Assertion Consumer Service URL as the default value when no identifier is provided in the request.

  4. Click OK to add the provider to the entity.
  5. Continue configuring the entity by selecting another option from the View menu or click Save to complete the configuration.
