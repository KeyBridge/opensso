Web Service Provider

The Web Service Provider agent profile describes the configuration that is used for securing web service requests from a web service provider. The names of the web services must be unique to each web service provider.

General

The following General attributes define basic web service provider properties:

Group

The Group mechanism allows you to define a collection of similar types of agents. The group must defined before including the particular agent into a collection.

Password

Defines the password for the web service provider attribute.

Password Confirm

Confirm the password.

Status

Defines whether web service provider agent will participate in securing outbound service requests. By default, it is set to Active, meaning it will participate.

Universal Identifier

Lists the basic LDAP properties, that uniquely define the web service provider agent.

Security

The following attributes define web service provider security attributes:

Security Mechanism

Defines the type of security credential that is used to secure the web services provider request. You can choose from the following security types:

  • Anonymous — The anonymous security mechanism contains no security credentials.

  • LibertyBearerToken

  • LibertySAMLToken

  • LibertyX509Token — Uses the Libery-defined X509 cerficiate to secure the web provider request.

  • SAML-HolderOfKey — Uses the SAML 1.1 assertion type Holder-Of-Key for securing the request.

  • SAML-SenderVouches — Uses the SAML 1.1 assertion type Sender Vouches for securing the request.

  • SAML2–HolderOfKey — Uses the SAML 2.0 assertion token type Holder-Of-Key for securing the request.

  • SAML2–SenderVouches — Uses the SAML 2.0 assertion token type Sender Vouches for securing the request.

  • UserNameToken — Uses a user name token to secure the web provider client requests.

  • UserNameToken-Plain — Uses a user name token with a clear text password for securing web provider requests.

  • X509Token — Uses the X509 certificate to secure the web provider requests.

Preserve Security Headers in Message

When enabled, this attribute defines that the SOAP security headers are preserved to the web service provider for further processing.

Liberty Service Type URN

The URN (Universal Resource Name) describes a Liberty service type that the web service provider will use for service lookups.

Credential for User Token

The attribute represents the username/password shared secrets that are used by the web service provider to generate a Username security token.

SAML Configuration

The following attributes configure the Security Assertion Markup Language (SAML) for the web service provider:

SAML Attribute Mapping

This configuration represents a SAML attribute that needs to be generated during assertion creation by the Security Token Service. The same mapping is used for the web service provider to populate it to the SSOToken.

SAML NameID Mapper Plugin

Defines the NameID mapper plug-in class that is used for SAML account mapping.

SAML Attributes Namespace

Defines the name space used for generating SAML attributes.

Include Memberships

If enabled, this attribute defines that the principal's membership must be included as a SAML attribute.

Singing and Encryption

The following attributes define signing and encryption configuration for web provider security:

Is Response Singed

When enabled, the web services provider signs the response using a given token type.

Is Response Encrypted

When enabled, the web services response will be encrypted.

Is Request Signature Verified

When enabled, the web services request signature is verified.

Is Request Header Encrypted

When enabled, the web services client security header will be encrypted.

Is Request Decrypted

When enabled, the web service provider request will be decrypted.

Key Store

The following attributes configure the keystore to be used for certificate storage and retrieval:

Public Key Alias of Web Service client

This attribute defines the public certificate key alias that is sued to encrypt the web service response.

Private Key Alias

This attribute defines the private certificate key alias that is used to sign/decrypt the web service request.

Key Storage Usage

This configuration defines whether to use the default keystore, or a custom keystore. The following values must be defined for a custom key store:

  • Location of Key Store

  • Password of Key Store

  • Password of Key

End Points

The following attributes define web service endpoints:

Web Service End Point

This attribute defines a web service end point to which the web service client is making a request.
