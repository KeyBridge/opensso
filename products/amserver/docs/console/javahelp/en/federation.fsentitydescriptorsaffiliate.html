To Add Members to an Affiliate Entity

After creating an affiliate entity, you may add members to the affiliation. The starting point is Entities under the Federation interface.

Note - You may also modify defined Common Attributes from this screen.

  1. Click on the name of a configured affiliate entity to modify its profile.

    The entity's profile page is displayed.

  2. Select Affiliate from the View menu.
  3. Select any of the available entities to assign to the affiliation.
  4. Modify values for the Common Attributes.

    Common Attributes contain values that generally define the identity provider itself.

    Name

    Type a new URI for the affiliation, if applicable. The current value of this attribute is the URI assigned to the Affiliate Name attribute when you created the entity. Any new value must be unique among affiliate names.

    Owner

    Type a new URI to define a different owner or parent operator for the affiliation. The current value of this attribute is the URI assigned to the Affiliate Owner attribute when you created the entity. The new value for this attribute is a URI that would match the value defined in the Entity Name attribute of the entity to which the new owner/provider belongs. Any new value must be unique among affiliate names.

    Note - Defining a provider as the Affiliate Owner does not automatically include it as a member of the affiliation. If an owner is also a member, the URI must be defined here as the owner and within the affiliate entity itself as a member.

    Signing Key: Key Alias

    Certificates are stored in a Java keystore file. Each certificate in the file is mapped to an alias used to fetch it. Type the key alias associated with the certificate used to sign requests and responses.

    Encryption Key: Key Alias

    Certificates are stored in a Java keystore file. Each certificate in the file is mapped to an alias used to fetch it. Type the key alias associated with the provider's security certificate.

    Encryption Key: Key Size

    Type the length for keys used by the web service consumer when interacting with another entity.

    Encryption Key: Encryption Method

    Type a URI pointing to the encryption method preference.

  5. Click Save.
