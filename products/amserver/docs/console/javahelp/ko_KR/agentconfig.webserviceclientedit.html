웹 서비스 클라이언트 속성

웹 서비스 클라이언트 에이전트 프로필은 웹 서비스 클라이언트에서 보낸 아웃바운드 웹 서비스 요청을 보호하기 위해 사용하는 구성을 설명합니다. 웹 서비스의 이름은 각 웹 서비스 클라이언트에 대해 고유해야 합니다.

상태

웹 서비스 클라이언트 에이전트가 아웃바운드 서비스 요청 보호에 참여할지 여부를 정의합니다. 기본적으로 활성으로 설정되며, 이는 아웃바운드 서비스 요청 보호에 참여함을 의미합니다.

범용 식별자

보안

다음 속성은 웹 서비스 클라이언트 보안 속성을 정의합니다.

보안 메커니즘

웹 서비스 클라이언트 요청을 보호하는 데 사용할 보안 자격 증명 유형을 정의합니다. 다음 보안 유형 중에서 선택할 수 있습니다.

  • 익명 — 익명 보안 메커니즘에 보안 자격 증명이 포함되지 않습니다.

  • LibertyDiscoverySecurity — 리버티 기반 보안 토큰을 사용하여 요청을 보호합니다.

  • SAML-HolderOfKey — SAML 1.1 명제 유형 Holder-Of-Key를 사용하여 요청을 보호합니다.

  • SAML-SenderVouches — SAML 1.1 명제 유형 Sender Vouches를 사용하여 요청을 보호합니다.

  • SAML2–HolderOfKey — SAML 2.0 명제 토큰 유형 Holder-Of-Key를 사용하여 요청을 보호합니다.

  • SAML2–SenderVouches — SAML 2.0 명제 토큰 유형 Sender Vouches를 사용하여 요청을 보호합니다.

  • STSSecurity — 웹 서비스 클라이언트가 보안 토큰 서비스에서 보안 토큰을 가져옵니다.

  • UserNameToken — 사용자 이름 토큰을 사용하여 웹 서비스 클라이언트 요청을 보호합니다.

  • UserNameToken-Plain — 일반 텍스트 비밀번호가 있는 사용자 이름 토큰을 사용하여 웹 서비스 요청을 보호합니다.

  • X509Token — X509 인증서를 사용하여 웹 서비스 요청을 보호합니다.

STS 구성

웹 서비스 클라이언트가 보안 토큰 서비스(STS)를 사용하는 경우 이 속성을 사용합니다. 이 구성은 STS 서비스에 대한 요청을 보호하는 데 사용하는 STS 에이전트 프로필 목록을 설명합니다.

검색 구성

이 속성은 검색 서비스 보안에 웹 서비스 클라이언트를 사용하는 경우에 사용됩니다. 이 구성은 검색 서비스에 대한 요청을 보호하는 데 사용하는 검색 에이전트 프로필 목록을 설명합니다.

사용자 인증 필요

이 속성은 서비스 클라이언트의 보호된 페이지에 사용자가 액세스하려면 인증을 받아야 함을 정의합니다.

메시지의 보안 헤더 보존

이 속성을 사용하면 향후 처리를 위해 웹 서비스 클라이언트가 SOAP 보안 헤더를 보존합니다.

리버티 서비스 유형 URN

URN(Universal Resource Name)은 웹 서비스 클라이언트가 서비스 조회에 사용할 리버티 서비스 유형을 설명합니다.

사용자 토큰의 자격 증명

이 속성은 웹 서비스 클라이언트가 사용자 이름 보안 토큰을 생성하는 데 사용할 사용자 이름/비밀번호 공유 비밀을 나타냅니다.
