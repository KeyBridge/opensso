부트스트랩용 자원 오퍼링 구성

부트스트랩에는 하나의 자원 오퍼링만 허용됩니다. 기본적으로 이 오퍼링에는 검색 서비스 관련 정보가 들어 있습니다. 자원 오퍼링이 이미 정의되어 있는 경우 해당 이름을 눌러 속성을 수정할 수 있습니다. 이름 옆에 있는 상자를 선택하여 기존 자원 오퍼링을 삭제할 수도 있습니다. 새 자원 오퍼링을 구성하려면 [새로 만들기]를 누릅니다.

  1. [새로 만들기]를 눌러 새 매퍼 속성을 표시하거나 구성된 매퍼 이름을 눌러 매퍼를 수정합니다.

  2. 자원 오퍼링에 대한 선택 사항인 설명을 입력합니다.

  3. 다음 정보를 기반으로 서비스 식별자 속성의 값을 제공합니다.

    서비스 유형

    이 속성은 자원 오퍼링이 구현하는 서비스 유형을 정의하는 URI를 값으로 가집니다. 예를 들면 urn:liberty:disco:2003-08과 같습니다.

    주 – 이 URI를 서비스의 추상 WSDL 설명에 대한 targetNamespace URI와 같게 하는 것이 좋습니다.

    공급자 아이디

    이 속성은 서비스 인스턴스 공급자의 URI를 값으로 가집니다. 예를 들면 http://daiquiri.sun.com:80/amserver/Liberty/disco와 같습니다.

    서비스 설명

    이 속성은 개별 프로토콜 종점에서 실행 중인 웹 서비스를 정의합니다. 서비스 인스턴스에 대한 정보는 다양한 컨텍스트에서 통신이 가능해야 합니다. 예를 들어, 이 사양에 정의된 검색 서비스는 자원 오퍼링(각각 서비스 인스턴스 설명을 포함)을 열거하는 Identity 서비스입니다. 서비스 설명 구성을 참조하십시오.

  4. 다음 정보를 기반으로 자원 오퍼링 옵션의 속성 값을 제공합니다.

    옵션

    서비스에 자원 오퍼링에 사용할 수 있는 옵션이 없는 경우 이 상자를 선택합니다. 이 속성은 특정 데이터나 작업을 특정 자원 오퍼링에 사용할 수 있는지 여부에 대한 힌트를 잠재적 요청자에게 제공합니다. 예를 들어 집 연락처 정보를 사용할 수 있는지 여부를 나타내는 옵션을 지정할 수 있습니다.

    옵션 목록

    이 속성은 서비스 인스턴스의 옵션 목록을 포함합니다. 이 옵션은 URI로 정의됩니다. 가능한 URI 집합은 일반적으로 서비스 유형에 따라 표준화됩니다.

  5. 다음 정보를 기반으로 지시문을 사용합니다.

    설명 아이디 참조: GenerateBearerToken

    모든 지원되는 지시문에는 설명 참조가 포함될 수 있습니다. 지시문에 대해 아이디 참조 속성을 사용할 수 없는 경우에는 자원 오퍼링에서 제공하는 모든 인증 기법에 이 지시문이 적용됩니다. 지시문에 대해 아이디 참조 속성을 사용할 수 있는 경우에는 지시문이 연결된 인증 기법을 참조하는 아이디 참조 목록이 지시문에 포함되어 있어야 합니다. 이 속성이 있는 경우 지시문은 아이디 참조 목록에서 참조되는 설명에만 적용해야 합니다. 이 속성은 특정 지시문이 특정 보안 기법과 호환되지 않을 경우 유용할 수 있습니다.

  7. 검색 서비스 페이지에서 [저장]을 눌러 서비스 구성을 완료합니다.
