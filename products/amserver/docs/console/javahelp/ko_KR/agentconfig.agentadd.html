새 에이전트를 만들려면

새 에이전트를 만드는 것은 새 에이전트 프로필을 만드는 것입니다. Federated Access Manager 관리 콘솔에서 이 작업을 수행할 수 있습니다. 이 작업은 웹 에이전트, J2EE 에이전트 및 2.2 에이전트에 적용됩니다. 그러나 일부 개별 단계는 2.2 에이전트에 적용되지 않습니다.

  1. [구성] 탭의 [에이전트]를 누릅니다.
  2. 필요한 경우 적절한 에이전트 유형에 해당하는 탭을 선택합니다.
    • 웹 에이전트:

      [웹] 탭은 웹 에이전트용이며, 기본적으로 선택됩니다. 다른 선택은 필요하지 않습니다.

    • J2EE 에이전트:

      [J2EE] 탭은 J2EE 에이전트용입니다. 해당하는 경우 이 탭을 누릅니다.

    • 2.2 에이전트:

      [2.2 에이전트] 탭은 정책 에이전트 2.2 소프트웨어 집합에서 에이전트용으로 사용됩니다. 해당하는 경우 이 탭을 누릅니다.

  3. [에이전트] 섹션에서 [새로 만들기]를 누릅니다.
  4. [이름] 필드에 새 에이전트 프로필의 이름을 입력합니다.
  5. 비밀번호를 입력하고 확인합니다.

    주의 – 이 비밀번호는 agentadmin 프로그램을 실행하여 에이전트를 설치할 때 지정한 에이전트 프로필 비밀번호 파일에 입력한 비밀번호와 동일해야 합니다.

  6. 적용 가능한 구성을 로컬 또는 중앙 집중화됨(2.2 에이전트에 적용되지 않음) 중에 선택합니다.

    또한 로컬 구성을 선택한 경우에는 이 에이전트와 관련된 속성을 에이전트의 파일에서만 편집할 수 있으며 콘솔에서는 편집할 수 없습니다.

  7. [서버 URL] 필드에 Federated Access Manager 서버 URL(2.2 에이전트에 적용되지 않음)을 입력합니다.

    예를 들면 다음과 같습니다.

    http://fam.example.com:8080/fam
  8. [에이전트 URL] 필드에 해당 에이전트 응용 프로그램 agentapp의 URL을 입력합니다(2.2 에이전트에 적용되지 않음).

    예를 들면 다음과 같습니다.

    http://fam.example.com:8090/agentapp
  9. [만들기]를 누릅니다.

    콘솔이 에이전트 프로필을 만들고 새 에이전트 프로필에 대한 링크가 있는 에이전트(WebAgent, J2EEAgent 또는 2.2_Agent) 페이지를 다시 표시합니다.

    에이전트 프로필을 추가로 구성하려면 이 링크를 눌러 에이전트 편집 페이지를 표시합니다.
