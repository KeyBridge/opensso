SecurID

이 모듈은 RSA ACE/서버 소프트웨어 및 RSA SecurID 인증자를 사용하는 인증을 허용합니다. Linux 또는 Solaris x86 플랫폼에서는 SecurID 인증 모듈을 사용할 수 없으므로 이 두 플랫폼에서 등록, 구성 및 사용해서는 안됩니다. SecurID 인증 모듈은 Solaris에서만 사용할 수 있습니다. SecurID 인증 속성은 영역 속성입니다. 속성은 다음과 같습니다.

  • ACE/서버 구성 경로

  • 도우미 구성 포트

  • 도우미 인증 포트

  • 인증 수준

ACE/서버 구성 경로

SecurID ACE/서버 sdconf.rec 파일이 있는 디렉토리를 지정합니다. 기본적으로 이 파일은 /opt/ace/data에 있습니다. 이 필드에 다른 디렉토리를 지정한 경우에는 이 디렉토리가 있어야 SecurID 인증을 시도할 수 있습니다.

도우미 구성 포트

SecurID 도우미가 시작될 때 SecurID 도우미 인증 포트 속성에 포함된 구성 정보를 '수신'하는 포트를 지정합니다. 기본값은 58943입니다.

이 속성을 변경하는 경우 AMConfig.properties 파일의 securidHelper.ports 항목도 변경하고 Access Manager를 다시 시작해야 합니다. AMConfig.properties 파일의 항목은 SecurID 도우미 인스턴스에 대한 포트 목록입니다. 각 포트는 공백으로 구분되어 있습니다. 다른 sdconf.rec 파일을 가진 다른 ACE/서버와 통신하는 각 영역에 대해 별도의 SecurID 도우미가 있어야 합니다.

도우미 인증 포트

영역의 SecurID 인증 모듈이 인증 요청을 '수신'하도록 SecurID 도우미 인스턴스를 구성하는 포트를 지정합니다. 이 포트 번호는 SecurID 또는 UNIX 인증을 사용하는 모든 영역에서 고유해야 합니다. 기본값은 57943입니다.

인증 수준

인증 수준은 각 인증 방법에 대해 별도로 설정됩니다. 이 값은 인증 기법을 어느 정도 신뢰할 수 있는지를 나타냅니다. 사용자가 인증되고 나면 해당 세션의 SSO 토큰에 이 값이 저장됩니다. 사용자가 액세스하려는 응용 프로그램에 이 SSO 토큰이 제공되면 응용 프로그램은 저장된 값을 사용하여 해당 수준이 사용자에게 액세스를 허가할 만큼 충분한지 여부를 확인합니다. SSO 토큰에 저장된 인증 수준이 필요한 최소값을 충족하지 않을 경우 응용 프로그램은 더 높은 인증 수준을 가진 서비스를 통해 다시 인증을 받으라는 메시지를 사용자에게 표시할 수 있습니다. 기본값은 0입니다.

주 – 지정된 인증 수준이 없을 경우 SSO 토큰에는 핵심 인증 속성인 기본 인증 수준에 지정된 값이 저장됩니다.
