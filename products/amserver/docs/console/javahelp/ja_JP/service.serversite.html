サーバーおよびサイト

「サーバーおよびサイト」設定属性を使用すると、配備全体のサイトとサーバーの設定を集中管理することができます。

少なくとも 2 台の異なるホストサーバーに、複数 (2 つ以上) の Access Manager インスタンスを配備することができます。たとえば、1 台のサーバーに 2 つのインスタンスを配備し、もう 1 台のサーバーに 3 番目のインスタンスを配備することができます。または、すべてのインスタンスを別々のサーバーに配備することもできます。また、配備に必要な場合は、Federated Access Manager インスタンスをセッションフェイルオーバーモードに設定することもできます。

1 つ以上のロードバランサが、クライアント要求をさまざまな Federated Access Manager インスタンスに経路指定します。配備の要件 (たとえば、ラウンドロビンまたは負荷平均値の使用) に従って各ロードバランサを設定し、Federated Access Manager インスタンス間で負荷を分散します。ロードバランサを使用すると配備が簡単になり、クライアントとバックエンドの Access Manager サーバー間のファイアウォールなどの問題も解決されます。Access Manager の配備にハードウェアロードバランサまたはソフトウェアロードバランサを使用することができます。すべての Access Manager インスタンスが、同じ Directory Server にアクセスします。
