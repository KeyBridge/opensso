新規エージェントを作成する

新規エージェントの作成は、新規エージェントプロファイルの作成とも呼ばれます。この作業は Federated Access Manager 管理コンソールで実行できます。この作業は、Web エージェント、J2EE エージェント、および 2.2 エージェントに適用できます。ただし、個々の手順の一部は、2.2 エージェントには適用されません。

  1. 「設定」タブの「エージェント」をクリックします。
  2. 必要に応じて、適切なエージェントタイプのタブを選択します。
    • Web エージェント:

      「Web」タブは Web エージェント用で、デフォルトで選択されています。これ以上の選択は必要ありません。

    • J2EE エージェント:

      「J2EE」タブは J2EE エージェント用です。該当する場合は、このタブをクリックします。

    • 2.2 エージェント

      「2.2 エージェント」タブは、ポリシーエージェント 2.2 ソフトウェアセットのエージェント用です。該当する場合は、このタブをクリックします。

  3. 「エージェント」セクションの「新規」をクリックします。
  4. 「名前」フィールドに新しいエージェントプロファイルの名前を入力します。
  5. パスワードを入力し、確認します。

    注意 – このパスワードは、agentadmin プログラムを実行してエージェントをインストールする際に指定する、エージェントプロファイルパスワードファイルに入力するパスワードと一致する必要があります。

  6. 該当する設定として「ローカル」または「集中化」を選択します (2.2 エージェントは対象外)。

    さらに、ローカルを選択した場合、このエージェントに関連する属性はエージェント内のファイルでのみ編集できます。コンソールから編集することはできません。

  7. 「サーバー URL」フィールドに Federated Access Manager サーバーの URL を入力します (2.2 エージェントは対象外)。

    次に例を示します。

    http://fam.example.com:8080/fam
  8. 「エージェント URL」フィールドにエージェントアプリケーション agentapp の URL を入力します (2.2 エージェントは対象外)。

    次に例を示します。

    http://fam.example.com:8090/agentapp
  9. 「作成」をクリックします。

    コンソールでエージェントプロファイルが作成され、そのエージェントプロファイルへのリンクを含んだエージェント (WebAgent、J2EEAgent、または 2.2_Agent) ページが再度表示されます。

    エージェントプロファイルに対して追加の設定を行うには、このリンクをクリックして「エージェントの編集」ページを表示します。
