Web サービス

Web サービスモジュールは、Liberty Alliance Project の Identity Web Services Framework (ID-WSF) 仕様の Access Manager における実装です。ID-WSF は、Liberty ID-FF (Identity Federation Framework) に基づいて、連携ネットワーク環境でのアイデンティティーデータに対して検索、更新、またはアクションを実行する Web サービスを構築するための仕様を提供しています。この仕様には、カレンダサービス、ウォレットサービス、アラートサービスなどの Web サービスの構築に必要な技術的要素の概要がまとめられています。

注 – 「アイデンティティーデータ」には、姓、名、自宅住所、勤務先住所、および緊急連絡先を定義する属性が含まれていますが、これらに限定されるわけではありません。

Web サービスは、インターネット全体での分散コンピューティングの土台となっています。Web サービスコンシューマ (WSC) は Web サービスを見つけ、要求を出すことによってそのサービスが提供する処理を引き出します。Web サービスプロバイダ (WSP) は、要求に基づいて Web サービスを実装するアプリケーションです。WSP は WSC と同じ Java™ 仮想マシンに配置することも、何千マイルも離れた場所に配置することもできます。WSC が WSP からアイデンティティー属性を取得する必要がある場合、特定の属性が格納されている場所を見つけるために WSC は最初にディスカバリサービスに接続する必要があります。この情報が返されると、WSC は WSP (個人プロファイルサービスなど) に接続して必要な属性を取得します。

Access Manager コンソールの Web サービスモジュールは、次のサービスを設定するためのインタフェースを提供します。

  • 「Liberty 個人プロファイルサービス」

  • 「ディスカバリサービス」

  • 「SOAP バインドサービス」

  • 「認証 Web サービス」
