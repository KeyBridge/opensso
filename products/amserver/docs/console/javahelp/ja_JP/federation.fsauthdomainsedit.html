認証ドメインプロファイルを変更する

次の手順に従って、既存の認証ドメインの設定済み「一般」属性を編集するか、既存の認証ドメインにプロバイダを追加します。連携インタフェースの「認証ドメイン」から作業を始めます。

  1. 設定済みの認証ドメインのプロファイルを変更したりプロバイダを追加したりするには、名前をクリックします。

    「認証ドメインを編集」ページが表示されます。

  2. 認証ドメインの次の「一般」属性に新規の値を入力するか、または既存の値を編集します。
    名前

    この属性の静的な値は、認証ドメインの作成時に指定した名前です。

    説明

    この属性の値は認証ドメインの説明です。適切な場合には、すでに入力されている説明を変更することもできます。

    ライターサービス URL

    この属性は、共通ドメインからの Cookie を書き込むサービスの場所を指定します。URL の形式は http://common-domain-host:port/common/writer です。

    リーダーサービス URL

    この属性は、共通ドメインからの Cookie を読み込むサービスの場所を指定します。URL の形式は http://common-domain-host:port/common/transfer です。

    状態

    デフォルトの状態は「アクティブ」です。「非アクティブ」を選択すると、認証ドメイン内の通信が無効になります。

  3. プロバイダを認証ドメインに追加するには、「追加」をクリックします。

    信頼パートナーを選択するページが表示されます。

  4. 1 つ以上の利用可能なプロバイダを選択し、「追加」矢印をクリックして追加します。

    表示される一覧には、作成されプロバイダが設定されたエンティティーの名前が入ります。詳細は、「プロバイダを認証ドメインに追加する」を参照してください。

  5. 「了解」をクリックして、認証ドメインに追加したプロバイダを保存します。

    「認証ドメイン」プロファイル画面に戻ります。

  6. 「保存」をクリックして、操作を完了します。
