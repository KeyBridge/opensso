認証ドメインプロファイルを変更する

既存の認証ドメインの「一般」属性を編集したり、認証ドメインにプロバイダを追加したりするには、次の手順に従います(「プロバイダを認証ドメインに追加する」を参照)。連携モジュールの「認証ドメイン」設定画面から始めます。

手順

  1. 設定済みの認証ドメインのプロファイルを変更したりプロバイダを追加したりするには、名前をクリックします。

  2. 次の情報に基づいて、認証ドメインの「一般」属性の値を編集します。

    名前

    この必須の属性は、認証ドメインの名前を定義します。

    説明

    この属性には認証ドメインの説明が含まれています。

    ライタサービス URL

    この属性は、共通ドメインからの Cookie を書き込むサービスの場所を指定します。URL の形式は http://common-domain-host:port/common/writer です。

    リーダサービス URL

    この属性は、共通ドメインからの Cookie を読み込むサービスの場所を指定します。URL の形式は http://common-domain-host:port/common/transfer です。

    状態

    デフォルトの状態は「アクティブ」です。「非アクティブ」を選択すると、認証ドメイン内の通信が無効になります。

  3. プロバイダを認証ドメインに追加するには、「追加」をクリックします。

  4. 「利用可能なプロバイダ」リストからプロバイダを 1 つ以上選択し、「追加」をクリックします。

    注 - 「プロバイダを認証ドメインに追加する」を参照してください。

  5. 「了解」をクリックして、認証ドメインに追加したプロバイダを保存します。

  6. 「保存」をクリックして、操作を完了します。
