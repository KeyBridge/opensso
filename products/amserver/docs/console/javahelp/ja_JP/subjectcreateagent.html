エージェントを作成または変更する

  1. 「エージェント」タブをクリックします。

  2. 「新規」をクリックします。

  3. 次のフィールドに値を入力します。

    ID: エージェントの名前またはアイデンティティーを入力します。これは、エージェントが Access Manager へのログイン時に使用する名前です。マルチバイトの名前は入力できません。

    パスワード: エージェントのパスワードを入力します。このパスワードは、エージェントが LDAP 認証時に使用するパスワードとは異なっている必要があります。

    パスワードの確認: パスワードを確認します。

    デバイスの状態: エージェントのデバイスの状態を入力します。これが「アクティブ」に設定された場合、エージェントは、Access Manager に対して認証や通信を行えます。これが「非アクティブ」に設定された場合、エージェントは、Access Manager への認証を行えません。

  4. 「了解」をクリックします。

  5. エージェントの作成が完了すると、さらに次のフィールドを編集できます。

    説明: エージェントの簡単な説明を入力します。たとえば、エージェントのインスタンス名や保護対象アプリケーションの名前などを入力できます。

    エージェントキー値: エージェントのプロパティーをキーと値のペアで設定します。このプロパティーは、Access Manager がユーザーに関する証明情報表明の要求をエージェントから受信する際に使用されます。現在のところ、有効になるプロパティーは 1 つだけであり、その他のプロパティーはすべて無視されます。次の形式を使用します。

    agentRootURL=http://server_name:port/
