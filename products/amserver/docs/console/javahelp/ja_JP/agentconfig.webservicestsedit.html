セキュリティートークンサービス

セキュリティートークンサービスは、セキュリティートークンの発行と管理を提供する Web サービスです。つまり、要求されていない場合でも暗号化されたセットで、セキュリティーステートメントやクレームを頻繁に作成します。これらのステートメントは、セキュリティートークンサービスが直接確認できる証拠を受け取るか、信頼している発行局からセキュリティートークンを受け取ることに基づいています。信頼を表明するために、サービスは STS によって発行された 1 つまたは一連のセキュリティートークンを提供することで、一連のクレームを表明する権利を証明することができます。または、独自の信頼ステートメントを含んだセキュリティートークンを発行することができます (セキュリティートークンの形式によっては、これは単に再発行または共同署名となる場合もあることに注意)。これは、信頼の仲介の基礎を形成します。

一般

次の一般属性は、セキュリティートークンサービスの基本的なプロパティーを定義します。

グループ

「グループ」メカニズムを使用すると、類似のエージェントタイプのコレクションを定義できます。特定のエージェントをコレクションに追加する前に、グループを定義しておく必要があります。

パスワード

セキュリティートークンサービス属性のパスワードを定義します。

パスワードの確認

パスワードを確認します。

ステータス

セキュリティートークンサービスエージェントがアウトバウンドのサービス要求のセキュリティー保護に参加するかどうかを定義します。デフォルトでは Active に設定されるため、エージェントが参加することになります。

汎用識別子

セキュリティートークンサービスエージェントを一意に定義する、基本的な LDAP プロパティーのリスト。

セキュリティー

次の属性は、セキュリティートークンサービスのセキュリティー属性を定義します。

セキュリティーメカニズム

セキュリティートークンサービスの要求をセキュリティー保護するために使用する、セキュリティー証明情報の種類を定義します。次のセキュリティーの種類を選択できます。

  • Anonymous — 匿名セキュリティーメカニズムにはセキュリティー証明情報はありません。

  • LibertyDiscoverySecurity — Liberty ベースのセキュリティートークンを使用して要求をセキュリティー保護します。

  • SAML-HolderOfKey — SAML 1.1 表明タイプ Holder-Of-Key を使用して要求をセキュリティー保護します。

  • SAML-SenderVouches — SAML 1.1 表明タイプ Sender Vouches を使用して要求をセキュリティー保護します。

  • SAML2–HolderOfKey — SAML 2.0 表明トークンタイプ Holder-Of-Key を使用して要求をセキュリティー保護します。

  • SAML2–SenderVouches — SAML 2.0 表明トークンタイプ Sender Vouches を使用して要求をセキュリティー保護します。

  • STSSecurity — セキュリティートークンサービスエージェントがセキュリティートークンサービスからセキュリティートークンを取得することを定義します。

  • UserNameToken — ユーザー名トークンを使用してセキュリティートークンサービスの要求をセキュリティー保護します。

  • UserNameToken-Plain — ユーザー名トークンと平文パスワードを使用してセキュリティートークンサービス要求をセキュリティー保護します。

  • X509Token — X509 証明書を使用してセキュリティートークンサービスエージェントの要求をセキュリティー保護します。

STS 設定

この属性は、セキュリティートークンサービスエージェントがセキュリティートークンサービス (STS) を使用する場合に有効になります。この設定は、STS サービスへの要求をセキュリティー保護するために使用される、STS エージェントプロファイルのリストを記述します。

メッセージにセキュリティーヘッダーを保存する

この属性が有効になっている場合は、以降の処理に使用できるように SOAP セキュリティーヘッダーがセキュリティートークンサービスエージェントに保持されます。

ユーザートークンの資格情報

この属性は、セキュリティートークンサービスエージェントがユーザー名セキュリティートークンを生成するために使用する、ユーザー名/パスワード共有シークレットを表します。

署名と暗号化

次の属性は、セキュリティートークンサービスに関する署名と暗号化の設定を定義します。

リクエストには署名が必要

これが有効になっている場合、セキュリティートークンサービスエージェントは特定のトークンタイプを使用して要求に署名します。

リクエストヘッダーの暗号化が必要

これが有効になっている場合は、セキュリティートークンサービスエージェントのセキュリティーヘッダーが暗号化されます。

リクエストの暗号化が必要

これが有効になっている場合は、セキュリティートークンサービスの要求が暗号化されます。

応答の署名の確認が必要

これが有効になっている場合は、セキュリティートークンサービス応答の署名が検証されます。

応答の復号化が必要

これが有効になっている場合は、セキュリティートークンサービスの応答が復号化されます。

キーストア

次の属性は、証明書の格納と検索に使用するキーストアを設定します。

Web サービスプロバイダの公開鍵のエイリアス

この属性は、Web サービス要求の暗号化に使用される証明書公開鍵エイリアスを定義します。

非公開鍵のエイリアス

この属性は、Web サービス応答の署名/復号化に使用される証明書非公開鍵エイリアスを定義します。

キーストアの使用法

この設定は、デフォルトのキーストアを使用するかカスタムのキーストアを使用するかを定義します。カスタムキーストアを使用する場合は、次の値を定義する必要があります。

  • キーストアの場所

  • キーストアのパスワード

  • キーのパスワード

エンドポイント

次の属性は、Web サービスエンドポイントを定義します。

Web サービスエンドポイント

この属性は、Web サービスクライアントが要求を行う Web サービスエンドポイントを定義します。
