国際化設定

国際化設定サービスには、さまざまなロケールや文字セット用に Access Manager を設定するためのグローバル属性が含まれます。属性は次のとおりです。

  • 「各ロケールでサポートされる文字セット」

  • 「文字セットのエイリアス」

  • 「自動生成される共通名の形式」

各ロケールでサポートされる文字セット

この属性は各ロケールでサポートされる文字セットを一覧表示します。これにより、ロケールと文字セット間のマッピングが示されます。形式は次のとおりです。

「新たにサポートする文字セット」を追加するには、「追加」をクリックして次のパラメータを定義します。

ロケール

追加する新しいロケール。詳細については、「サポートされている言語ロケール」を参照してください。

サポートされる文字セット

指定したロケールでサポートされる文字セットを入力します。文字セットはセミコロンで区切ります。たとえば、charset=charset1;charset2;charset3;...;charsetn のようにします。

既存のサポートされる文字セットを編集するには、「サポートされる文字セット」の表でその名前をクリックします。操作が完了したら、「了解」をクリックします。

文字セットのエイリアス

この属性では、応答を送信するのに使用されるコードセット名 (IANA 名にマッピングされる) が一覧表示されます。これらのコードセット名は Java のコードセット名と一致する必要はありません。現在、Java の文字セットと IANA の文字セットを相互にマッピングするためのハッシュ表が存在しています。

「新規文字セットのエイリアス」を追加するには、「追加」をクリックして次のパラメータを定義します。

MIME 名

IANA マッピング名。たとえば、Shift_JIS のように指定します。

Java 名

IANA の文字セットにマッピングする Java 文字セット。

既存の「文字セットのエイリアス」を編集するには、表でその名前をクリックします。操作が完了したら、「了解」をクリックします。

自動生成される共通名の形式

この表示オプションで、異なるロケールおよび文字セットに適応した名前を自動的に生成する方法を定義できます。デフォルトの構文は次のとおりです。実際に名前を表示する際に、定義内に含めたコンマやスペースもそのまま表示されることに注意してください。

en_us = {givenname} {initials} {sn}

たとえば、uid (11111) と ユーザー (User One) の新しい名前形式を中国語の文字セットで表示する場合、次のように定義します。

zh = {sn}{givenname}({uid})

結果は次のように表示されます。

OneUser (11111)
