Federated Access Manager データストアの種類

Federated Access Manager のデータストア選択画面で作成できるデータストアの種類は次のとおりです。

Federated Access Manager リポジトリプラグイン

このタイプのデータストアは Sun Java System Directory Server インスタンス内に存在し、そこに Federated Access Manager 情報ツリーが格納されます。このタイプのデータストアは、ロールやサービスのクラスなど LDAP version 3 の仕様に含まれない Directory Server 機能を使用します。また、Federated Access Manager の以前のバージョンとの互換性があります。

属性の定義については、「Federated Access Manager リポジトリの属性」を参照してください。

Active Directory

このタイプのデータストアは、LDAP version 3 仕様に基づいて、Microsoft Active Directory インスタンスにアイデンティティーデータを書き込みます。

属性の定義については、「Active Directory の属性」を参照してください。

フラットファイルリポジトリ

このリポジトリでは、別途にデータストアを作成することなく、Federated Access Manager のローカルインストールインスタンスのフラット DIT 構造内にデータとアイデンティティーを保存することができます。これは一般に、概念展開のテストまたは検証に使用されます。

属性の定義については、「フラットファイルリポジトリの属性」を参照してください。

汎用 LDAPv3

このタイプのデータストアは、アイデンティティーデータを任意の LDAPv3 準拠データベースに書き込むことができます。使用している LDAPv3 データベースが持続検索をサポートしない場合は、キャッシュ機能を利用できません。

属性の定義については、「汎用 LDAPv3 属性」を参照してください。

Federated Access Manager スキーマを含んだ Sun Directory Server

このタイプのデータストアは Sun Java System Directory Server インスタンス内に存在し、そこに Federated Access Manager 情報ツリーが格納されます。これは、豊富な設定属性によるデータストアの高度なカスタマイズが可能な点で Federated Access Manager リポジトリプラグインとは異なります。

属性の定義については、「Federated Access Manager スキーマ属性を含んだ Sun Directory Server」を参照してください。
