Web サービスプロバイダ

Web サービスプロバイダエージェントプロファイルは、Web サービスプロバイダからの Web サービス要求をセキュリティー保護するために使用される設定を記述します。Web サービスの名前は、Web サービスプロバイダごとに一意でなければなりません。

一般

次の一般属性は、Web サービスプロバイダの基本的なプロパティーを定義します。

グループ

「グループ」メカニズムを使用すると、類似のエージェントタイプのコレクションを定義できます。特定のエージェントをコレクションに追加する前に、グループを定義しておく必要があります。

パスワード

Web サービスプロバイダ属性のパスワードを定義します。

パスワードの確認

パスワードを確認します。

ステータス

Web サービスプロバイダエージェントがアウトバウンドのサービス要求のセキュリティー保護に参加するかどうかを定義します。デフォルトでは Active に設定されるため、エージェントが参加することになります。

汎用識別子

Web サービスプロバイダエージェントを一意に定義する、基本的な LDAP プロパティーのリスト。

セキュリティー

次の属性は、Web サービスプロバイダのセキュリティー属性を定義します。

セキュリティーメカニズム

Web サービスプロバイダの要求をセキュリティー保護するために使用する、セキュリティー証明情報の種類を定義します。次のセキュリティーの種類を選択できます。

  • Anonymous — 匿名セキュリティーメカニズムにはセキュリティー証明情報はありません。

  • LibertyBearerToken

  • LibertySAMLToken

  • LibertyX509Token — Libery で定義された X509 証明書を使用して Web プロバイダの要求をセキュリティー保護します。

  • SAML-HolderOfKey — SAML 1.1 表明タイプ Holder-Of-Key を使用して要求をセキュリティー保護します。

  • SAML-SenderVouches — SAML 1.1 表明タイプ Sender Vouches を使用して要求をセキュリティー保護します。

  • SAML2–HolderOfKey — SAML 2.0 表明トークンタイプ Holder-Of-Key を使用して要求をセキュリティー保護します。

  • SAML2–SenderVouches — SAML 2.0 表明トークンタイプ Sender Vouches を使用して要求をセキュリティー保護します。

  • UserNameToken — ユーザー名トークンを使用して Web プロバイダクライアントの要求をセキュリティー保護します。

  • UserNameToken-Plain — ユーザー名トークンと平文パスワードを使用して Web プロバイダの要求をセキュリティー保護します。

  • X509Token — X509 証明書を使用して Web プロバイダの要求をセキュリティー保護します。

認証連鎖

定義 —

メッセージにセキュリティーヘッダーを保存する

この属性が有効になっている場合、以降の処理で使用できるように SOAP セキュリティーヘッダーを Web サービスプロバイダに保持するように定義されます。

Liberty サービスタイプ URN

URN (Universal Resource Name) は、Web サービスプロバイダがサービス検索に使用する Liberty サービスタイプを記述します。

ユーザートークンの資格情報

この属性は、Web サービスプロバイダがユーザー名セキュリティートークンを生成するために使用する、ユーザー名/パスワード共有シークレットを表します。

非公開鍵のタイプ

定義 —

SAML 設定

次の属性は、Web サービスプロバイダの SAML (Security Assertion Markup Language) を設定します。

SAML 属性マッピング

この属性は、セキュリティートークンサービスが表明の作成時に生成する必要のある SAML 属性を表します。Web サービスプロバイダがこれを SSOToken に挿入するときにも、同じマッピングが使用されます。

SAML NameID マッパープラグイン

SAML アカウントマッピングに使用する NameID マッパープラグインクラスを定義します。

SAML 属性名前空間

SAML 属性の生成に使用するネームスペースを定義します。

包含メンバーシップ

この属性が有効になっている場合は、主体のメンバーシップを SAML 属性として含める必要があります。

署名と暗号化

次の属性は、Web プロバイダのセキュリティーに関する署名と暗号化の設定を定義します。

応答には署名が必要

これが有効になっている場合、Web サービスプロバイダは特定のトークンタイプを使用して応答に署名します。

応答の暗号化が必要

これが有効になっている場合は、Web サービスの応答が暗号化されます。

リクエストの署名の確認が必要

これが有効になっている場合は、Web サービス要求の署名が検証されます。

リクエストヘッダーの暗号化が必要

これが有効になっている場合は、Web サービスクライアントのセキュリティーヘッダーが暗号化されます。

リクエストの復号化が必要

これが有効になっている場合は、Web サービスプロバイダの要求が復号化されます。

キーストア

次の属性は、証明書の格納と取り出しに使用するキーストアを設定します。

Web サービスクライアントの公開鍵のエイリアス

この属性は、Web サービス応答の暗号化に使用される証明書公開鍵エイリアスを定義します。

非公開鍵のエイリアス

この属性は、Web サービス要求の署名/復号化に使用される証明書非公開鍵エイリアスを定義します。

キーストアの使用法

この設定は、デフォルトのキーストアを使用するかカスタムのキーストアを使用するかを定義します。カスタムキーストアを使用する場合は、次の値を定義する必要があります。

  • キーストアの場所

  • キーストアのパスワード

  • キーのパスワード

エンドポイント

次の属性は、Web サービスエンドポイントを定義します。

Web サービスエンドポイント

この属性は、Web サービスクライアントが要求を行う Web サービスエンドポイントを定義します。
