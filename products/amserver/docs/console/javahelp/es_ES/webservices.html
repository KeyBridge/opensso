Servicios Web

El módulo Servicios Web es la implementación de Access Manager de la especificaciones Marco de servicios Web de identidades Liberty (ID-WSF, Identity Web Services Framework) de Liberty Alliance Project. ID-WSF se basa en el Marco de federación de identidades Liberty (ID-FF) proporcionando especificaciones para crear servicios Web que recuperan, actualizan o realizan una acción en los datos de identidad en un entorno de red federado. Las especificaciones definen los componentes técnicos necesarios para crear servicios Web como, por ejemplo, el servicio de calendario, de cartera o un servicio de alertas.

Nota – Entre los datos de identidad se incluyen, pero sin limitarse a, los atributos que definen el nombre, los apellidos, el domicilio, la dirección de la empresa y un contacto de emergencia.

Los servicios Web son la base del cálculo distribuido en Internet. Un consumidor de servicios Web (WSC) localiza un servicio Web e invoca las operaciones que proporciona mediante la realización de una solicitud. Un proveedor de servicios Web (WSP, Web Service Provider) es la aplicación que implementa el servicio Web en función de la solicitud; puede encontrarse en la misma máquina virtual de Java™, como WSC, o a mucha distancia. Cuando un WSC debe recuperar los atributos de identidad desde un WSP, primero debe ponerse en contacto con un servicio de detección para encontrar la ubicación en la que se almacenan determinados atributos. Cuando se devuelve esta información, WSC se pone en contacto con WSP (por ejemplo, un servicio de perfil personal) para recuperar los atributos necesarios.

El módulo Servicios Web de la consola de Access Manager proporciona una interfaz para configurar los siguientes elementos:

  • Servicio de perfil personal de Liberty

  • Servicio de detección

  • Servicio de enlace de SOAP

  • Servicio Web de autenticación
