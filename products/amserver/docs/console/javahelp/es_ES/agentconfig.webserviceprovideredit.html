Proveedor de servicios web

El perfil de agente del proveedor de servicios web describe la configuración utilizada para proteger las solicitudes de servicios web de un proveedor de servicios web. Los nombres de los servicios web deben ser exclusivos para cada proveedor de servicios web.

General

Los siguientes atributos generales definen las propiedades básicas del proveedor de servicios web:

Grupo

El mecanismo de grupo permite definir un conjunto de tipos de agentes similares. Debe definirse el grupo antes de incluir un agente específico en un conjunto.

Password (Contraseña)

Define la contraseña del atributo de proveedor de servicios web.

Confirmar contraseña

Confirme la contraseña.

Estado

Define si el agente del proveedor de servicios web participará en la protección de las solicitudes de servicios salientes. Se establece de forma predeterminada en Active, lo que implica que el agente participará.

Identificador universal

Enumera las propiedades de LDAP básicas, que definen de forma exclusiva al agente del proveedor de servicios web.

Seguridad

Los siguientes atributos definen los atributos de seguridad del proveedor de servicios web:

Mecanismo de seguridad

Define el tipo de credenciales de seguridad que se utilizará para proteger la solicitud del proveedor de servicios web. Puede elegir entre los siguientes tipos de seguridad:

  • Anonymous: el mecanismo de seguridad anónimo no contiene ninguna credencial de seguridad.

  • LibertyBearerToken

  • LibertySAMLToken

  • LibertyX509Token: utiliza el certificado X509 definido por Liberty para proteger la solicitud del proveedor web.

  • SAML-HolderOfKey: utiliza el tipo de aserción de SAML 1.1 "Holder-Of-Key" (Contenedor de clave) para proteger la solicitud.

  • SAML-SenderVouches: utiliza el tipo de aserción de SAML 1.1 "Sender Vouches" (Confirmaciones del remitente) para proteger la solicitud.

  • SAML2–HolderOfKey: utiliza el tipo de testigo de aserción de SAML 2.0 "Holder-Of-Key" (Contenedor de clave) para proteger la solicitud.

  • SAML2–SenderVouches: utiliza el tipo de testigo de aserción de SAML 2.0 "Sender Vouches" (Confirmaciones del remitente) para proteger la solicitud.

  • UserNameToken: utiliza un testigo de nombre de usuario para proteger las solicitudes de cliente del proveedor web.

  • UserNameToken-Plain: utiliza un testigo de nombre de usuario con una contraseña de texto sin formato para proteger las solicitudes del proveedor web.

  • X509Token: utiliza el certificado X509 para proteger las solicitudes del proveedor web.

Cadena de autenticación

Definición –

Conservar encabezados de seguridad del mensaje

Si se habilita, este atributo permite conservar los encabezados de seguridad SOAP en el proveedor de servicios web para su posterior procesamiento.

URN de tipo de servicio de Liberty

El URN (Nombre de recursos universal) describe el tipo de servicio de Liberty que utilizará el proveedor de servicios web para la búsqueda de servicios.

Credencial del testigo de usuario

Este atributo representa los secretos compartidos de nombre de usuario/contraseña que utiliza el proveedor de servicios web para generar un testigo de seguridad de nombre de usuario.

Definición –

Configuración de SAML

Los siguientes atributos configuran el Lenguaje de marcado de aserción de seguridad (SAML) para el proveedor de servicios web:

Asignación de atributos de SAML

Esta configuración representa un atributo de SAML que debe generarse durante la creación de aserciones por parte del servicio de testigo de seguridad. El proveedor de servicios web utiliza la misma asignación para rellenarla en el testigo de SSO.

Complemento del asignador de Id. de nombre de SAML

Define la clase de complemento del asignador de Id. de nombre utilizada para la asignación de cuentas de SAML.

Nombre de espacio de los atributos de SAML

Define el nombre de espacio utilizado para generar los atributos de SAML.

Incluir miembros

Si se habilita, este atributo define si se deben incluir los miembros del elemento principal como atributos de SAML.

Firma y cifrado

Los siguientes atributos definen la configuración de firma y cifrado para la protección del proveedor web:

Se ha firmado la respuesta

Si se habilita, el proveedor de servicios web firma la respuesta mediante un determinado tipo de testigo.

Se ha cifrado la respuesta

Si se habilita, se cifrará la respuesta de los servicios web.

Se ha verificado la firma de la solicitud

Si se habilita, se verificará la firma de la solicitud de servicios web.

Se ha cifrado el encabezado de la solicitud

Si se habilita, se cifrará el encabezado de seguridad de cliente de los servicios web.

Se ha descifrado la solicitud

Si se habilita, se descifrará la solicitud del proveedor de servicios web.

Almacén de claves

Los siguientes atributos configuran el almacén de claves que se utilizará para el almacenamiento y la recuperación de los certificados:

Alias de clave pública del cliente de servicios web

Este atributo define el alias de clave pública del certificado que se ha utilizado para cifrar la respuesta del servicio web.

Alias de clave privada

Este atributo define el alias de clave privada del certificado que se ha utilizado para firmar/descifrar la solicitud de servicios web.

Uso del almacén de claves

Esta configuración define si se debe utilizar el almacén de claves predeterminado o uno personalizado. Deben definirse los siguientes valores para un almacén de claves personalizado:

  • Ubicación del almacén de claves

  • Contraseña del almacén de claves

  • Contraseña de la clave

Puntos finales

Los siguientes atributos definen los puntos finales del servicio web:

Punto final del servicio web

Este atributo define el punto final del servicio web en el que el cliente de servicios web realiza una solicitud.
