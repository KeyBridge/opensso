Para configurar un conjunto de conexiones: ejemplo

En el siguiente ejemplo se muestra cómo configurar un grupo de conexión para Web Server y MySQL 4.0:

  1. En la consola de Web Server, cree un grupo de conexión JDBC con los siguientes atributos:

    Nombre de conjunto

    samplePool

    Nombre de clase de origen de datos

    com.mysql.jdbc.jdbc2.optional.MysqlDatacSource

    Nombre de servidor

    Nombre del servidor mySQL.

    puerto

    Número de puerto en el que se va a ejecutar el servidor mySQL.

    usuario

    Nombre de usuario de la contraseña de la base de datos:

    contraseña

    La contraseña del usuario.

    Nombre de base de datos

    El nombre de la base de datos.

    Nota – El archivo jar que contiene la clase de origen de datos y la clase de controlador JDBC que se mencionan en los pasos siguientes debe agregarse a la ruta de la clase de aplicación

  2. Configure los recursos de JDBC. En la consola de Web Server, cree un recurso de JDBC con los siguientes atributos:

    Nombre JNDI

    jdbc/samplePool

    Nombre de conjunto

    samplePool

    Recurso de datos habilitado

    activado

  3. Agregue las siguientes líneas al archivo sun-web.xml de la aplicación:

    <resource-ref>
      <res-ref-name>jdbc/mySQL</res-ref-name>
      <jndi-name>jdbc/samplePool</jndi-name>
</resource-ref>

  4. Agregue las siguientes líneas al archivo web.xml de la aplicación:

    <resource-ref>
       <description>mySQL Database</description>
       <res-ref-name>jdbc/mySQL</res-ref-name>
       <res-type>javax.sql.DataSource</res-type>
       <res-auth>Container</res-auth>
</resource-ref>

  5. Una vez que haya completado la configuración, el valor de este atributo pasará a ser java:comp/env/jdbc/mySQL.
