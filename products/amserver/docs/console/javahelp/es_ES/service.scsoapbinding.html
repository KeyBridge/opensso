Servicio de enlace de SOAP

El Servicio de enlace de SOAP es una capa de transporte para enviar y recibir mensajes de SOAP. En el proceso del Servicio de enlace de SOAP, un servicio de identidad llama a la interfaz de programación de aplicación del cliente para crear un mensaje y enviarlo a la dirección URL de punto final de SOAP; de hecho, un servlet de receptor de SOAP. El servlet de receptor de SOAP recibe el mensaje, verifica la firma y crea un segundo mensaje. A continuación, el servlet de receptor de SOAP invoca al administrador de solicitudes para enviar este segundo mensaje al servicio de identidades correspondiente y obtener una respuesta. El servicio de identidades procesa el segundo mensaje, genera una respuesta y la envía al servlet del receptor de SOAP. El receptor de SOAP, a su vez, envía la respuesta al servicio de identidades para su procesamiento.

Se pueden configurar los siguientes atributos globales del Servicio de enlace de SOAP para la implementación.

  • Lista de administradores de solicitudes

  • Autenticador de servicio Web

  • Mecanismos de autenticación admitidos
