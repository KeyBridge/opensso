Políticas de referencia

Es posible que un administrador deba delegar las decisiones y las definiciones de políticas de un dominio en otra organización. (Las decisiones de políticas de un recurso también pueden delegarse en otros productos de políticas). Una política de referencia controla esta delegación de políticas tanto para la creación como la evaluación de políticas, y está formada por una o varias reglas y una o varias referencias.

El servicio de configuración de directivas contiene un atributo global denominado Referencias de alias de organización. Este atributo permite crear directivas en subdominios sin necesidad de crear directivas de referencia en el nivel superior o el dominio principal. Sólo se pueden crear directivas para proteger los recursos HTTP o HTTPS cuyo nombre de host completo coincida con el dominio o el alias de DNS del dominio. De forma predeterminada, este atributo se define como No.
