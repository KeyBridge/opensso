Lista de administradores de solicitudes

El Servicio de enlace de SOAP proporciona la interfaz RequestHandler para procesar el mensaje de solicitud y devolver una respuesta. La interfaz debe implementarse en el servidor por parte de cada servicio de identidad basado en Liberty que utilice el Servicio de enlace de SOAP. El atributo âLista de administradores de solicitudesâ guarda información sobre las clases de implementación de los servicios Web que implementa el administrador de solicitudes. La lista de administradores de solicitudes muestra entradas que contienen pares clave-valor separados por una línea vertical como en: key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService . Los parámetros se explican en la siguiente tabla.

Parámetro

Encabezado

key

El parámetro key requerido es la última parte de la ruta del URI a un punto final de SOAP. El punto final de SOAP en Access Manager el servlet del receptor de SOAP. El URI para el receptor de SOAP es: protocol:// host:port/deploy-uri/Liberty/key Si define disco como clave, la ruta del URI al punto final de SOAP para el Servicio de detección correspondiente será: protocol://host:port /amserver/Liberty/disco. Distintos clientes de servicios utilizan distintas claves al conectarse al receptor de SOAP.

class

Este parámetro class requerido especifica el nombre de la clase de implementación del administrador de solicitudes para el servicio de identidades concreto. Por ejemplo: class=com.example.identity.liberty.ws.disco.DiscoveryService .

Para agregar un nuevo administrador de solicitudes, consulte Para configurar un administrador de solicitudes.
