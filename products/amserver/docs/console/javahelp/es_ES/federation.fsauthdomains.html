Dominios de autenticación

Un dominio de autenticación es una federación de un número cualquiera de proveedores de servicios (y, al menos, un proveedor de identidades) con los que las entidades principales pueden tramitar negocios en un entorno seguro y aparentemente perfecto. Para crear y rellenar un dominio de autenticación, debe crear primero una entidad que albergue los metadatos (la información de configuración que define una determinada arquitectura de servicios de identidades) para cada proveedor que será miembro del dominio de autenticación. A continuación, configure y guarde el dominio de autenticación. Por último, para agregar una entidad (un proveedor configurado) al dominio de autenticación, edite las propiedades de la entidad.

Nota – Para obtener más información sobre las entidades, consulte Entidades: proveedor o afiliado.

Todos los dominios de autenticación configurados se muestran en Dominios de autenticación. A continuación se muestran distintas tareas asociadas a los dominios de autenticación:

  • Para crear un dominio de autenticación nuevo

  • Para modificar un perfil de dominio de autenticación

  • Para agregar proveedores a un dominio de autenticación

  • Para eliminar un perfil de dominio de autenticación

Nota – Los miembros de un dominio de autenticación deben haber establecido anteriormente un acuerdo operativo de círculo de confianza basado en las directrices del modelo de confianza de Liberty.
