Entidades: proveedor o afiliado

Una entidad puede configurarse con metadatos (información de configuración que define una determinada arquitectura de servicios de identidades) para un proveedor de identidades o un proveedor de servicios individual, o para cada uno de ellos. Por el contrario, una entidad también puede configurarse como una afiliación, es decir, un grupo seleccionado de proveedores de cualquiera de los dos tipos. Tanto las entidades de proveedores como de afiliación pueden configurarse en la consola de Access Manager.

Nota – Se puede crear una entidad, pero no estará disponible para asignarla a un dominio de autenticación hasta que se hayan agregado los proveedores.

Los dos tipos de entidades se describen en la siguiente tabla.

Tabla 1 Tipo de entidades

Entidad de proveedores

Una entidad de proveedores contiene los metadatos para los proveedores individuales de cada tipo. Todos los proveedores de identidades y servicios deben configurarse en una entidad de proveedores para poder asociarlos a un dominio de autenticación o seleccionarlos para incluirlos en una entidad de afiliación. Mediante los atributos proporcionados en la consola de Access Manager, un proveedor de identidades o un proveedor de servicios individual, o una de cada tipo puede definirse en una entidad de proveedores. Un perfil de entidad de proveedores contiene los siguientes grupos de atributos:

  • General Attributes

  • Identity Provider

  • Service Provider

Affiliate Entity

Una afiliación configurada permite al usuario realizar la federación con un grupo de sitios asociados. Por ejemplo, cuando un proveedor de servicios emite una solicitud de autenticación para un usuario en nombre de una afiliación, y se ha protegido la autenticación, el usuario puede realizar un inicio de sesión único con todos los miembros de la afiliación. La afiliación se forma sin tener en cuenta los límites de un dominio de autenticación.

Una entidad de afiliación contiene los metadatos que definen el grupo de una o varias de las entidades de proveedores que conforman la aplicación. No contiene la información de configuración de ningún proveedor, sólo la información de configuración del propio afiliado. Un perfil de entidad de afiliación contiene los siguientes grupos de atributos:

  • General Attributes

  • Affiliate

Nota – Los miembros de una afiliación puede llamar a servicios como miembro de la afiliación o individualmente.

Todos las entidades configuradas de ambos tipos se muestran en Entidades. A continuación se muestran distintas tareas asociadas con las entidades:

  • Para crear una entidad de proveedores o una entidad de afiliación

  • Para modificar los atributos generales de una entidad de proveedores o una entidad de afiliación

  • Para agregar un proveedor de identidades a una entidad de proveedores

  • Para agregar un nuevo proveedor de identidades alojado a una entidad de proveedores

  • Para agregar un nuevo proveedor de identidades remoto a una entidad de proveedores

  • Para agregar un proveedor de servicios a una entidad de proveedores

  • Para agregar un nuevo proveedor de servicios alojado a una entidad de proveedores

  • Para agregar un nuevo proveedor de servicios remoto a una entidad de proveedores

  • Para modificar atributos de un proveedor de identidades alojado en una entidad de proveedores

  • Para modificar atributos de un proveedor de identidades remoto en una entidad de proveedores

  • Para modificar atributos de un proveedor de servicios alojado en una entidad de proveedores

  • Para modificar atributos de un proveedor de servicios remoto en una entidad de proveedores

  • Para agregar una persona de contacto a un proveedor en una entidad de proveedores

  • Para modificar el perfil de una persona de contacto en una entidad de proveedores

  • Para agregar miembros a una entidad de afiliación
