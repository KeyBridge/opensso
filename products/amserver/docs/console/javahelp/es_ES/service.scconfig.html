Atributos de configuración

La página de configuración contiene todos los atributos para configurar los servicios predeterminados de Access Manager. Los atributos que componen un servicio de Access Manager se clasifican en los siguientes tipos (algunos servicios pueden pertenecer a más de un tipo):

Global: se aplican en toda la configuración de Access Manager. No se pueden aplicar a usuarios, roles o dominios, ya que el propósito de los atributos globales es personalizar la aplicación de Identity Server.

Dominio: los atributos de dominio sólo se asignan a dominios. No se asocia ninguna clase de objeto con los atributos de dominio. Los atributos que se muestran en los servicios de autenticación se definen como atributos de dominio porque la autenticación se lleva a cabo en el nivel de dominio y no en el nivel de árbol o usuario.

Dinámico: se asignan a roles o dominios configurados en Access Manager. Cuando se asigna un rol a un usuario o se crea un usuario en un dominio, el atributo dinámico se convierte en una característica del usuario.

Usuario: se asignan directamente a cada usuario. No se heredan de roles o dominios y, normalmente, son diferentes para cada usuario.

Las propiedades de configuración que puede modificar son las siguientes:

  • Autenticación

  • Propiedades de la consola

  • Propiedades globales

  • Propiedades del sistema
