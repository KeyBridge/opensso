Autorizador

Antes de procesar una solicitud, el Servicio de perfil personal de Liberty verificará la autorización del WSC que realiza la solicitud. Para ello, se pueden llevar a cabo dos niveles de comprobación de la autorización:

  1. ¿Dispone la entidad emisora de la solicitud de autorización para acceder a la información del perfil del recurso solicitado?

  2. ¿Se ha publicado el recurso solicitado para el solicitante?

Las autorizaciones tienen lugar mediante un complemento del Servicio de perfil personal de Liberty: una implementación de la interfaz com.sun.identity.liberty.ws.interfaces.Authorizer. Aunque es posible crear una implementación nueva, Federated Access Manager proporcionará la predeterminada: com.sun.identity.liberty.ws.idpp.plugin.IDPPAuthorizer . Este complemento define cuatro valores de acción de política para la solicitud y las operaciones de modificación:

  • Permitir

  • Denegar

  • Interactuar para consentir

  • Interactuar para valor

Los valores de recursos de las reglas son similares a las expresiones x-path definidas por el Servicio de perfil personal de Liberty.

/PP/CommonName/AnalyzedName/FN Query Interact for consent /PP/CommonName/* Modify Interact for value /PP/InformalName Query Deny
