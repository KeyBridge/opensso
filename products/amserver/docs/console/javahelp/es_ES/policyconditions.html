Condiciones

Una condición permite definir las restricciones de la política. Por ejemplo, si define una política para una aplicación de pago de nóminas, puede definir una condición en esta acción que limite el acceso a la aplicación durante únicamente determinadas horas. O también puede definir una condición que permita sólo esta acción si la solicitud se origina a partir de un determinado conjunto de direcciones IP o de una Intranet corporativa.

La condición puede utilizarse también para configurar diferentes políticas en distintos URI del mismo dominio. Por ejemplo, sólo se puede acceder a http://org.example.com/hr/*jsp mediante org.example.net desde las 9:00 a las 17:00. Para ello, se debe utilizar una condición de IP con una condición de tiempo. Además, al especificar el recurso de regla como http://org.example.com/hr/*.jsp, la directiva se aplicará a todos los JSP por debajo de http://org.example.com/hr , incluidos aquéllos que residen en los subdirectorios.

Nota – Los términos referencia, regla, recurso, asunto, condición y valor se corresponden con los elementos Referencia, Regla, Nombre de recurso, Asunto, Condición, Atributo y Valor en policy.dtd.
