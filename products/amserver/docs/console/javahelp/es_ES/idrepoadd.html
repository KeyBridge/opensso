Tipos de almacenes de datos de Federated Access Manager

En la pantalla de selección de almacenes de datos de Federated Access Manager puede crear los siguientes tipos de almacenes de datos:

Complemento de repositorio de Federated Access Manager

Este tipo de almacén de datos reside en la instancia de Sun Java System Directory Server y contiene el árbol de información de Federated Access Manager. Además, utiliza las funciones de Directory Server que no forman parte de la especificación LDAP versión 3 como, por ejemplo, los roles y la clase de servicio, y es compatible con las versiones anteriores de Federated Access Manager.

Para conocer las definiciones de atributos, consulte Atributos de repositorio de Federated Access Manager.

Active Directory

Este tipo de almacén de datos utiliza la especificación LDAP versión 3 para escribir datos de identidades en una instancia de Microsoft Active Directory.

Para obtener las definiciones de atributos, consulte Atributos de Active Directory.

Repositorios de archivos sin formato

Este repositorio permite almacenar datos e identidades en una estructura DIT sin formato en la instancia de instalación local de Federated Access Manager sin necesidad de crear un almacén de datos independiente. Normalmente se utiliza para la realización de pruebas o la implementación de prototipos.

Para obtener las definiciones de atributos, consulte Atributos de repositorio de archivos sin formato.

LDAPv3 genérico

Este tipo de almacén de datos permite escribir datos de identidades en cualquier base de datos compatible con LDAPv3. Si la base de datos LDAPv3 que utiliza no admite la búsqueda persistente, no podrá utilizar la función de almacenamiento en caché.

Para obtener las definiciones de atributos, consulte Atributos LDAPv3 genéricos.

Sun Directory Server con esquema de Federated Access Manager

Este tipo de almacén de datos reside en la instancia de Sun Java System Directory Server y contiene el árbol de información de Federated Access Manager. Se diferencia del complemento de repositorio de Federated Access Manager en que dispone un mayor número de atributos de configuración, lo que le permite personalizar de forma más eficaz el almacén de datos.

Para obtener las definiciones de atributos, consulte Sun Directory Server con atributos de esquema de Federated Access Manager.
