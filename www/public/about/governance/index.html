OpenSSO Governance Proposal - Draft

The OpenSSO project follows a simple governance structure. Decisions are made in public discussion groups by consensus. All decisions are made transparently and any member of the project may voice their concerns in the discussion.

Everyone is welcome to join the OpenSSO community and make contributions. OpenSSO members follow the code of conduct? and abide by the governance process. OpenSSO members may play one or more of the following roles in the community.

User

All OpenSSO project members are Users. Users use the software produced by the community. They file problem reports, enhancement requests and new feature request. They share their experiences in using the software with others in the community through forums, wiki and by publishing articles. Become a user by creating a user id and just joining the community. In opensso.java.net, request for observer role, which is equivalent to *User* role.

Developers

Developers are Users, who also contribute code or documentation changes to resolve issues, to make feature enhancements, or to incorporate new features. Developers do not have write access to the source tree. To become a developer, complete and sign Sun Contributor Agreement. Once SCA is registered, Users will be notified about the same. Please forward your registration note to sca@dev.opensso.java.net, if your role is not approved in two working days. In java.net, request for observer role. Observer role in opensso.java.net with SCA agreement signed is equivalent to Developer role.

Committers

Committers are Developers with write acess to the source tree. Committers review the patch submitted by the developers. Once the patch is approved by the committers and module owners, patch will be applied to the src tree by one of the committers. Most committers have write privileges to only part of the source tree that they are most familiar with. Developers are considered for committer role based on the quality of their prior contributions to the project. Expertise in one or more of the modules demonstrated through patches made to the src tree, contributions to the forums, and wiki. In java.net, request for developer role, which is equivalent to Committer.

Module Owners

Module owners are Committers, who are responsible for reviewing and maintaining the quality of the module. They are also responsible for merging patches, and new code from the development branch of the source tree into the stable branch. Module owners are also responsible for making sure that these contributions do not break the build and perform second level review as well as architecture review of changes proposed. Module owners are appointed by creating consensus among committers and module owners. Module owners are initially appointed by Sun. In java.net, request for developer role and content developer role.

Project Managers

This Project Managers are responsible for managing the entire project, helping to create policies by consensus that ensure global quality. Project managers are responsible for creating project branch of source tree. Branches are created for sustaining, major enhancements, and experiments. Project managers are also responsible for making any updates to governance policy. All decisions are made by driving consensus. In cases where consensus could not be reached, project managers make the final decision. Project Managers are appointed by Sun. In java.net, request for developer, content developer and project owner roles.
