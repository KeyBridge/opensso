Q: What is OpenSSO?

The Open Web Single Sign-On Project, also referred to as OpenSSO, is an open development effort based on the source code for Sun JavaTM System Access Manager, a core identity infrastructure product offered by Sun Microsystems, Inc. The goal of OpenSSO is to provide an extensible foundation for an identity services infrastructure that will facilitate single sign-on (SSO) for web applications hosted on web and application servers in the public domain.

Q: Why would Sun open source SSO?

Sun's goal is to improve developer access to this technology so that it can become a ubiquitous part of web security infrastructure. It is a given that companies are going to secure their internal web infrastructure using SSO technology, and the OpenSSO project gives developers the freedom to do that without adding software license costs to their budgets. The official Sun Microsystems answer to this FAQ can be found on the Sun web site at http://www.sun.com/software/products/identity/opensso/faq.xml.

Q: What are the benefits of open-sourcing software?

  • Allows developers to cooperate freely across time and distance with a minimum of legal restrictions.
  • Increases code quality and security.
  • Reduces deployment costs.
  • Increases the capacity for customization.
  • Encourages reusable software.
  • Enables a community where skill and merit matter.

Q: What are the benefits of OpenSSO?

The modules available in OpenSSO make up a common identity service, allowing for user authentication and single sign-on within a domain. Sun is providing this identity service to developers so they can incorporate it into each web or J2EE-based application and service they provide to internal employees and contractors. The availability of open source code that provides identity-based authentication and single sign-on capabilities will lead to broad adoption of the technology in the Java community.

Q: What comprises OpenSSO?

Sun Microsystems will provide the source for the following modules contained in Sun Java System Access Manager to the Java developer community on a free right to use basis:

  • Session Management
  • Policy
  • Console
  • Administration tools
  • Federation
  • Policy agents

Q: What are the objectives of the OpenSSO project?

  • To provide open access to an identity infrastructure source code.
  • To enable innovation to build the next generation of open network identity services.
  • To establish open XML-based file formats and language-independent component applicatin programming interfaces (API).

Q: How large is the OpenSSO community?

The OpenSSO project, for all intents and purposes, went live in July, 2006 so the development community is small. The expectation is that developers working on the open source code will greatly increase as news of the project spreads.

Q: What is the URL for the OpenSSO project?

The OpenSSO project is hosted at http://opensso.dev.java.net.

Q: What is available on the OpenSSO project site?

The OpenSSO project site contains the source code, nightly builds, the governance, documentation, forums, mailing lists, and links to relevant information. In addition, it contains instructions on how to become involved in this project.

Q: What license is used for the OpenSSO code base?

The OpenSSO code will be released under the Common Development and Distribution License (CDDL). Information about this license can be found at http://www.sun.com/cddl/ and http://www.opensource.org/licenses/cddl1.php.

Q: Is there any cost for using the OpenSSO source code?

No. The OpenSSO source code is free to use, free to modify and free to redistribute.

Q: Do I need to register to use the OpenSSO source code I downloaded from the site?

You do not need to register as a user on the java.net site in order to download the source code for OpenSSO. Nor is there a need to register specifically as a member of the OpenSSO project.

Q: What version of Sun Java System Access Manager has been open sourced?

Open sourced components are snapshots of the latest version of Access Manager under development, roughly corresponding to version 7.1. At this time, there are no plans to open source the code for previous versions of Access Manager.

Q: If I build the code for Access Manager, can I call it Access Manager?

No. You can not call what you build, or any modifications thereof, Access Manager. You can call it anything else, however, that is not already trademarked.

Q: How do I participate in the OpenSSO project?

The OpenSSO project follows a simple governance structure.

  • Decisions are made in public discussion groups.
  • Decisions are made by consensus, rather than voting.
  • Decisions are made transparently and any member of the project may voice their concerns.
Everyone is welcome to join the OpenSSO community and make contributions. OpenSSO members follow the code of conduct and abide by the governance process. OpenSSO members may play one or more of the following roles:
  • User
  • Developer
  • Committer
  • Module Owner
  • Project Manager
See the Governance for more details.

Q: Where do I start?

The best place to start is the OpenSSO home page where you can find links to the nightly builds, discussion groups, documentation, downloads, source code, white papers, and more. You can also find information in the FAQ Center.
